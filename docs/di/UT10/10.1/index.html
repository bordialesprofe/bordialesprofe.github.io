10.1 Evaluación y pruebas

Una vez desarrollada una aplicación, es necesario comprobar que todas sus funcionalidades funcionan correctamente. Durante el desarrollo pueden aparecer errores o comportamientos inesperados, por lo que antes de considerar finalizado un proyecto es importante realizar pruebas de forma organizada.

Las pruebas permiten verificar que la aplicación responde como se espera, detectar posibles fallos y asegurar que los cambios realizados no afectan negativamente a otras partes del sistema. De esta forma se mejora la calidad del software y se reduce la probabilidad de errores cuando la aplicación se utilice en un entorno real.

1️⃣ Introducción a las pruebas​

Las pruebas de software consisten en ejecutar una aplicación o parte de ella para comprobar si su comportamiento coincide con el resultado esperado.

Durante una prueba se realiza una acción concreta en la aplicación y se observa el resultado obtenido. Si el resultado coincide con lo que se esperaba, la prueba se considera correcta. En caso contrario, se detecta un posible error que debe revisarse.

Ejemplo: Salón de héroes

En la aplicación Salón de Héroes, una prueba sencilla podría consistir en comprobar que el registro de un nuevo héroe funciona correctamente.

Pasos de la prueba:

  1. Introducir el nombre del héroe.
  2. Seleccionar su clase.
  3. Guardar el héroe.

Resultado esperado: El héroe aparece correctamente en el listado de héroes registrados.

🔶 Por qué es importante probar una aplicación​

Las pruebas son una parte fundamental del desarrollo de software porque permiten detectar problemas antes de que la aplicación sea utilizada por los usuarios.

Entre sus principales ventajas se encuentran:

  • Detección temprana de errores. Permiten localizar fallos antes de que la aplicación se publique o se entregue a los usuarios.

  • Mejora de la calidad del software. Las pruebas ayudan a comprobar que todas las funcionalidades funcionan correctamente y que la aplicación responde de forma estable.

  • Mayor confianza en el funcionamiento de la aplicación. Cuando una aplicación ha sido probada adecuadamente, es más probable que funcione correctamente en diferentes situaciones.

Importancia de las pruebas

Detectar errores durante el desarrollo o antes de la publicación es mucho menos costoso que corregirlos cuando la aplicación ya está siendo utilizada por usuarios reales.

🔶 Cuándo se realizan las pruebas​

Las pruebas pueden realizarse en distintos momentos del desarrollo de una aplicación.

  • Durante el desarrollo. Es habitual comprobar cada funcionalidad mientras se está implementando para detectar errores de forma temprana.
  • Antes de la publicación. Cuando la aplicación está terminada, se realizan pruebas más completas para comprobar que todas las partes del sistema funcionan correctamente.
  • Después de realizar cambios. Si se modifica alguna funcionalidad, es recomendable volver a comprobar que el resto de la aplicación sigue funcionando correctamente.
Relación con el desarrollo del módulo

Las pruebas que se realicen en esta unidad se aplicarán sobre las aplicaciones desarrolladas durante el módulo, como el proyecto del Salón de Héroes.

2️⃣ Tipos de pruebas de software​

Las pruebas de software pueden realizarse de diferentes maneras según el aspecto de la aplicación que se quiera comprobar. No todas las pruebas persiguen el mismo objetivo: algunas verifican funcionalidades concretas, otras comprueban cómo interactúan distintos componentes o evalúan el rendimiento de la aplicación.

Conocer los principales tipos de pruebas permite seleccionar la estrategia más adecuada para evaluar el funcionamiento de una aplicación.

🔶 Pruebas unitarias​

Las pruebas unitarias verifican el funcionamiento de una unidad pequeña de código, normalmente una función, método o componente individual.

El objetivo es comprobar que cada parte del programa funciona correctamente de forma aislada.

Ejemplo: prueba unitaria

Se quiere comprobar que la función que calcula el nivel medio de los héroes devuelve el valor correcto.

Datos de prueba:

HéroeNivel
Arion10
Lyra20
Kael15

Resultado esperado: El nivel medio debe ser 15.

🛠Herramientas habituales

Existen herramientas que permiten automatizar las pruebas unitarias y comprobar automáticamente si una función o método devuelve el resultado esperado.

Algunas herramientas muy utilizadas son:

  • JUnit – framework de pruebas unitarias para aplicaciones Java.
  • pytest – muy utilizado para pruebas en Python.
  • Flutter test – framework integrado para realizar pruebas unitarias en aplicaciones Flutter.

Actualmente también existen herramientas de asistencia basadas en IA, como GitHub Copilot, que pueden ayudar a generar automáticamente casos de prueba unitarios a partir del código.

🔶 Pruebas funcionales​

Las pruebas funcionales verifican que las funcionalidades visibles para el usuario funcionan correctamente.

Se centran en comprobar que cada acción disponible en la aplicación produce el resultado esperado.

Ejemplo: prueba funcional

En Salón de Héroes, una prueba funcional podría consistir en comprobar que el registro de un nuevo héroe funciona correctamente.

Pasos de la prueba:

  • Introducir nombre del héroe.
  • Seleccionar la clase.
  • Guardar.

Resultado esperado: El héroe aparece en el listado de héroes registrados.

🛠Herramientas habituales

Las pruebas funcionales pueden automatizarse mediante herramientas que simulan las acciones de un usuario sobre la interfaz de la aplicación.

Entre las más conocidas se encuentran:

  • Selenium – una de las herramientas más utilizadas para automatizar pruebas en aplicaciones web.
  • Cypress – herramienta moderna orientada a pruebas de interfaces web.
  • Playwright – permite probar aplicaciones web en distintos navegadores de forma automatizada.

Algunas plataformas modernas como Testim utilizan inteligencia artificial para mantener los tests incluso cuando la interfaz de la aplicación cambia.

🔶 Pruebas de integración​

Las pruebas de integración comprueban que distintos componentes de la aplicación funcionan correctamente cuando trabajan juntos.

No se evalúan elementos aislados, sino la interacción entre ellos.

Ejemplo:prueba de integración

En Salón de Héroes se podría comprobar que:

  • un héroe creado en el formulario
  • se guarda correctamente
  • y aparece en la pantalla de listado.

Aquí intervienen varios componentes:

  • formulario
  • almacenamiento de datos
  • listado de héroes
🛠Herramientas habituales

Las pruebas de integración suelen realizarse utilizando herramientas que permiten comprobar la interacción entre distintos componentes del sistema.

Algunas herramientas utilizadas en este tipo de pruebas son:

  • JUnit – también permite realizar pruebas de integración en aplicaciones Java.
  • Postman / Newman – muy utilizados para comprobar el funcionamiento de APIs y servicios web.
  • Supertest – utilizado en aplicaciones Node.js para probar servicios y APIs.

Estas herramientas permiten verificar que los distintos módulos de la aplicación funcionan correctamente cuando trabajan conjuntamente.

🔶 Pruebas de regresión​

Las pruebas de regresión se realizan después de modificar el código para comprobar que los cambios no han generado nuevos errores en partes que antes funcionaban correctamente.

Ejemplo: prueba de regresión

Si se modifica el sistema de registro de héroes para añadir un nuevo campo (por ejemplo, el reino de origen), se deben repetir las pruebas anteriores para comprobar que:

  • el registro sigue funcionando
  • el listado sigue mostrando los héroes correctamente
🛠Herramientas habituales

Las pruebas de regresión suelen automatizarse para poder ejecutarlas cada vez que se realizan cambios en el código.

Entre las herramientas utilizadas destacan:

  • **Selenium Grid **– permite ejecutar pruebas automatizadas en distintos navegadores.
  • Cypress – permite repetir pruebas de forma automática tras cada cambio en la aplicación.
  • TestComplete – herramienta comercial para automatizar pruebas de regresión.

Algunas plataformas modernas como Mabl utilizan inteligencia artificial para detectar cambios en la interfaz y adaptar automáticamente las pruebas.

🔶 Pruebas de usabilidad​

Las pruebas de usabilidad evalúan la facilidad de uso de la aplicación desde el punto de vista del usuario.

Se analizan aspectos como:

  • claridad de la interfaz
  • facilidad para encontrar funciones
  • comprensión de los elementos de la aplicación
Ejemplo:prueba de usabilidad

En Salón de Héroes, se podría comprobar si un usuario nuevo es capaz de:

  • Encontrar el botón para registrar un héroe.
  • Introducir los datos correctamente.
  • Guardar el héroe sin dificultad.
🛠Herramientas habituales

Las pruebas de usabilidad suelen apoyarse en herramientas que permiten analizar cómo interactúan los usuarios con la aplicación.

Algunas herramientas utilizadas son:

  • Hotjar – permite analizar el comportamiento de los usuarios mediante mapas de calor y grabaciones de sesiones.
  • Maze – herramienta para realizar pruebas de usabilidad con usuarios reales.
  • Google Analytics – permite analizar el comportamiento general de los usuarios dentro de una aplicación o página web.

Estas herramientas ayudan a detectar dificultades en la navegación o problemas en la experiencia de uso.

🔶 Pruebas de rendimiento​

Las pruebas de rendimiento evalúan cómo se comporta la aplicación cuando maneja grandes cantidades de datos o muchos usuarios.

Se analizan aspectos como:

  • tiempo de respuesta
  • consumo de recursos
  • estabilidad del sistema
Ejemplo: prueba de rendimiento

En Salón de Héroes, se podría comprobar cómo funciona el listado de héroes cuando existen cientos o miles de registros.

🛠Herramientas habituales

Las pruebas de rendimiento permiten comprobar cómo se comporta una aplicación cuando recibe muchas peticiones o gestiona grandes cantidades de datos.

Algunas herramientas utilizadas son:

  • Apache JMeter – herramienta muy conocida para realizar pruebas de carga y estrés.
  • k6 – herramienta moderna para realizar pruebas de rendimiento en aplicaciones web.
  • Locust – permite simular múltiples usuarios accediendo simultáneamente a una aplicación.

Existen también plataformas como Dynatrace, que utilizan inteligencia artificial para detectar automáticamente problemas de rendimiento en aplicaciones.

Elegir el tipo de prueba adecuado

No todas las aplicaciones necesitan realizar todos los tipos de pruebas.
En proyectos pequeños suele ser suficiente combinar pruebas funcionales y pruebas de integración para comprobar el funcionamiento general del sistema.

3️⃣ Plan de pruebas​

Antes de comenzar a realizar pruebas sobre una aplicación es recomendable definir un plan de pruebas.

Un plan de pruebas es un documento en el que se especifica qué pruebas se van a realizar, cómo se van a ejecutar y qué resultados se esperan obtener.

Su objetivo es organizar el proceso de evaluación de la aplicación y asegurar que todas las funcionalidades importantes han sido comprobadas.

Un plan de pruebas permite:

  • organizar las pruebas que se van a realizar
  • identificar las funcionalidades que deben comprobarse
  • efinir los resultados esperados
  • registrar posteriormente los resultados obtenidos

De esta forma se garantiza que el proceso de pruebas se realiza de forma ordenada y sistemática.

🔶Elementos de un plan de pruebas​

Un plan de pruebas suele incluir diferentes elementos que permiten describir cada prueba de forma clara.

Entre los elementos más habituales se encuentran:

  • Identificador de la prueba. Permite identificar cada prueba de forma única.
  • Descripción de la prueba. Explica qué funcionalidad de la aplicación se va a comprobar.
  • Datos de entrada. Información que se utilizará durante la prueba.
  • Resultado esperado. Comportamiento que debería producir la aplicación si funciona correctamente.
  • Resultado obtenido. Resultado real obtenido tras ejecutar la prueba.
  • Estado de la prueba. Indica si la prueba ha sido superada o si se ha detectado algún error.
Plan de pruebas

Para comprobar el funcionamiento de la aplicación Salón de Héroes, se puede definir un plan de pruebas sencillo.

IDFuncionalidadDatos de entradaResultado esperado
P01Registrar héroeNombre: ArionEl héroe aparece en el listado
P02Registrar héroeNombre vacíoLa aplicación muestra un mensaje de error
P03Filtrar héroesClase: GuerreroSe muestran solo los héroes de esa clase

Este plan permite comprobar diferentes situaciones de uso de la aplicación.

🔶 Ventajas de utilizar un plan de pruebas​

Definir un plan de pruebas aporta varias ventajas durante el desarrollo de software.

Entre las más importantes se encuentran:

  • permite comprobar que todas las funcionalidades importantes han sido evaluadas
  • facilita detectar errores antes de publicar la aplicación
  • permite repetir las pruebas cuando se realizan cambios en el programa
  • ayuda a documentar el proceso de evaluación del software

En proyectos reales de desarrollo, los planes de pruebas forman parte de la documentación del proyecto y permiten garantizar la calidad del software.

4️⃣ Registro de resultados​

Una vez definido el plan de pruebas y ejecutadas las pruebas correspondientes, es necesario registrar los resultados obtenidos.

El registro de resultados permite documentar qué ha ocurrido durante cada prueba y comprobar si la aplicación funciona correctamente.

En este registro se indica:

  • qué prueba se ha ejecutado
  • cuál era el resultado esperado
  • cuál ha sido el resultado obtenido
  • si la prueba se ha superado o ha fallado

Esta información resulta muy útil para detectar errores en la aplicación y para realizar un seguimiento del proceso de pruebas.

Además, el registro de resultados permite repetir las pruebas en el futuro y comprobar si los errores detectados han sido corregidos.

🔶 Información registrada en las pruebas​

El registro de resultados suele incluir algunos campos básicos que permiten describir el resultado de cada prueba.

Entre los más habituales se encuentran:

  • Identificador de la prueba. Permite relacionar el resultado con la prueba definida en el plan de pruebas.
  • Resultado esperado. Comportamiento que debería producir la aplicación.
  • Resultado obtenido. Comportamiento real observado al ejecutar la prueba.
  • Estado de la prueba. Indica si la prueba ha sido superada (Correcto) o si se ha detectado un error (Fallido).
  • Observaciones. Comentarios adicionales sobre el comportamiento de la aplicación.
Ejemplo

Tras ejecutar las pruebas definidas en el plan de pruebas de la aplicación Salón de Héroes, se pueden registrar los resultados obtenidos.

IDResultado esperadoResultado obtenidoEstado
P01El héroe aparece en el listadoEl héroe aparece correctamenteCorrecto
P02Se muestra un mensaje de errorNo aparece ningún mensajeFallido
P03Se muestran solo héroes guerrerosEl filtro funciona correctamenteCorrecto

Este registro permite identificar rápidamente qué pruebas se han superado y en cuáles se han detectado errores.

🔶 Utilidad del registro de pruebas​

Registrar los resultados de las pruebas permite mejorar el proceso de evaluación del software.

Entre sus principales ventajas se encuentran:

  • facilita la detección de errores en la aplicación
  • permite realizar un seguimiento del proceso de pruebas
  • ayuda a comprobar si los errores han sido corregidos en versiones posteriores
  • permite documentar el funcionamiento de la aplicación antes de su publicación

En proyectos de desarrollo reales, estos registros forman parte de la documentación de calidad del software y ayudan a garantizar el correcto funcionamiento de la aplicación.
