10.2 Documentación

En el desarrollo de software no solo es importante que una aplicación funcione correctamente, sino también que pueda entenderse, utilizarse y mantenerse con facilidad. Para ello es necesario elaborar una documentación adecuada del proyecto.

La documentación permite explicar cómo funciona la aplicación, cómo debe utilizarse y cómo está organizada internamente. De esta forma, otras personas pueden comprender el proyecto, utilizarlo correctamente o continuar su desarrollo en el futuro.

En proyectos reales, la documentación forma parte fundamental del proceso de desarrollo y suele incluir diferentes tipos de documentos, como manuales de usuario, guías rápidas o documentación técnica del proyecto.

Además, una buena documentación facilita:

  • el uso correcto de la aplicación por parte de los usuarios
  • el mantenimiento y evolución del software
  • la incorporación de nuevos desarrolladores al proyecto
  • la comprensión de la estructura y funcionamiento del sistema

En esta sección se estudiarán los principales tipos de documentación utilizados en aplicaciones software y cómo elaborarlos de forma clara y organizada.

1️⃣ Documentación en el desarrollo de software​

La documentación forma parte fundamental del desarrollo de una aplicación. Permite explicar cómo funciona el software, cómo debe utilizarse y cómo está organizado el proyecto.

Una buena documentación facilita que otras personas puedan comprender la aplicación, utilizarla correctamente o continuar su desarrollo en el futuro.

En proyectos reales, la documentación se utiliza durante todo el ciclo de vida del software. No solo se crea al final del proyecto, sino que suele elaborarse y actualizarse a medida que el sistema evoluciona.

Además, una documentación clara permite:

  • explicar el funcionamiento de la aplicación a los usuarios
  • facilitar el mantenimiento del software
  • ayudar a otros desarrolladores a comprender el proyecto
  • registrar las decisiones tomadas durante el desarrollo

Por este motivo, la documentación forma parte de la calidad del software y suele incluir diferentes tipos de documentos según el objetivo que se persiga.

🔶 Qué es la documentación del software​

La documentación del software es el conjunto de documentos que describen el funcionamiento, la estructura y el uso de una aplicación.

Estos documentos pueden estar dirigidos a diferentes personas:

  • usuarios de la aplicación, que necesitan saber cómo utilizarla
  • desarrolladores, que necesitan comprender cómo está construido el sistema
  • administradores, que necesitan instalar o configurar el software

Dependiendo del público al que va dirigida, la documentación puede tener diferentes niveles de detalle y utilizar distintos formatos.

🔶 Tipos de documentación​

En el desarrollo de software suelen distinguirse varios tipos de documentación según su finalidad.

Los más habituales son:

  • Manual de usuario. Explica cómo utilizar la aplicación paso a paso.
  • Guía rápida. Presenta de forma resumida las funciones principales del sistema.
  • Documentación técnica. Describe la estructura del proyecto, los componentes de la aplicación y su funcionamiento interno.

Cada uno de estos documentos cumple una función diferente y suele elaborarse en distintas fases del desarrollo del proyecto.

2️⃣ Manual de usuario​

El manual de usuario es un documento que explica cómo utilizar una aplicación. Está dirigido principalmente a las personas que van a usar el software y necesitan comprender sus funcionalidades.

Su objetivo es guiar al usuario para que pueda utilizar correctamente la aplicación sin necesidad de conocer su funcionamiento interno.

Un manual de usuario suele describir las diferentes funciones del sistema y explicar paso a paso cómo realizar las acciones más habituales.

Para que resulte útil, el manual debe utilizar un lenguaje claro y sencillo, incluir ejemplos y, cuando sea necesario, capturas de pantalla que ayuden a comprender el funcionamiento de la aplicación.

🔶 Contenido de un manual de usuario​

Aunque el contenido puede variar según el tipo de aplicación, un manual de usuario suele incluir algunos apartados básicos.

Entre los más habituales se encuentran:

  • Introducción a la aplicación. Explica brevemente qué hace el sistema y cuál es su objetivo.
  • Descripción de la interfaz. Presenta las diferentes secciones o pantallas de la aplicación.
  • Instrucciones de uso. Explica cómo realizar las principales acciones dentro del sistema.
  • Resolución de problemas. Incluye posibles errores y cómo solucionarlos.

Organizar el manual de forma clara permite que los usuarios encuentren fácilmente la información que necesitan.

Ejemplo: fragmento de manual de usuario

En la aplicación Salón de Héroes, es posible añadir nuevos personajes al listado de héroes. Esta función permite guardar la información básica de cada héroe para poder consultarla posteriormente.

Añadir un héroe

  1. Abrir la aplicación Salón de Héroes.
  2. Acceder al formulario para añadir un nuevo héroe.
  3. Introducir el nombre del héroe.
  4. Seleccionar la clase correspondiente.
  5. Pulsar el botón Guardar.

Una vez completados estos pasos, el nuevo héroe aparecerá en el listado de héroes de la aplicación.

🛠Herramientas habituales

Existen diferentes herramientas que permiten crear manuales de usuario para aplicaciones.

Algunas de las más utilizadas son:

  • Microsoft Word o Google Docs, que permiten elaborar manuales con texto, imágenes y capturas de pantalla.
  • Markdown, utilizado para crear documentación sencilla y fácil de mantener.
  • Documentación web, mediante herramientas como Docusaurus o MkDocs, que permiten publicar manuales en formato web.

Estas herramientas facilitan la creación y organización de la documentación para que pueda consultarse fácilmente por los usuarios.

Además de las herramientas tradicionales, actualmente existen herramientas basadas en inteligencia artificial que pueden ayudar a generar documentación automáticamente a partir del código o de descripciones del sistema.

Algunos ejemplos son:

  • ChatGPT o GitHub Copilot, que pueden ayudar a redactar manuales de usuario o guías de uso a partir de la descripción de una aplicación.
  • Mintlify, que genera documentación automáticamente a partir del código fuente.
  • Document360 AI, que ayuda a crear bases de conocimiento y manuales técnicos.
  • Notion AI, que puede utilizarse para generar y organizar documentación de proyectos.

Estas herramientas permiten acelerar la creación de documentación, aunque siempre es recomendable revisar y adaptar el contenido para asegurar que la información es clara y correcta para los usuarios.

3️⃣ Guía rápida​

Una guía rápida es un documento breve que explica de forma resumida cómo utilizar las funciones principales de una aplicación.

A diferencia del manual de usuario, que suele ser más completo y detallado, la guía rápida se centra únicamente en las acciones más importantes que un usuario necesita conocer para empezar a utilizar el sistema.

Este tipo de documento suele utilizarse como material de consulta rápida, permitiendo a los usuarios recordar fácilmente cómo realizar determinadas acciones dentro de la aplicación.

Las guías rápidas suelen incluir:

  • instrucciones muy breves
  • pasos numerados
  • esquemas o capturas de pantalla
  • explicaciones sencillas y directas

Por este motivo, suelen utilizarse como complemento del manual de usuario.

🔶 Características de una guía rápida​

Las guías rápidas tienen algunas características que las diferencian de otros tipos de documentación.

Entre las más importantes se encuentran:

  • Brevedad. La información se presenta de forma muy resumida.
  • Organización clara. Se utilizan pasos numerados o listas para facilitar la lectura.
  • Contenido esencial. Solo se incluyen las funciones más importantes de la aplicación.
  • Facilidad de consulta. El usuario debe poder encontrar rápidamente la información que necesita.

Estas características hacen que la guía rápida sea especialmente útil para usuarios que ya conocen la aplicación y solo necesitan recordar cómo realizar alguna acción.

Ejemplo: fragmento guía rápida

Añadir un héroe

  1. Abrir la aplicación Salón de Héroes.
  2. Acceder al formulario de héroes.
  3. Introducir el nombre y la clase del héroe.
  4. Pulsar el botón Guardar.

Consultar héroes

  1. Acceder al listado de héroes.
  2. Utilizar el buscador o los filtros disponibles para localizar un personaje.

Esta guía rápida permite recordar las funciones principales de la aplicación sin necesidad de consultar el manual completo.

🛠Herramientas habituales

Las guías rápidas pueden elaborarse utilizando diferentes herramientas de documentación.

Entre las más utilizadas se encuentran:

  • Procesadores de texto, como Microsoft Word o Google Docs.
  • Herramientas de diseño, como Canva o Figma, que permiten crear guías visuales.
  • Markdown, utilizado para crear documentación sencilla dentro de proyectos de software.
  • Herramientas de documentación web, como Docusaurus o MkDocs.

Actualmente también existen herramientas basadas en inteligencia artificial que pueden ayudar a generar guías rápidas a partir de la descripción de una aplicación o del contenido de un manual de usuario (ver apartado Manual de usuario).

4️⃣ Documentación técnica​

La documentación técnica describe la estructura interna de una aplicación y proporciona información útil para los desarrolladores que trabajan en el proyecto.

A diferencia del manual de usuario o la guía rápida, que están dirigidos a los usuarios finales, la documentación técnica está orientada a personas que desarrollan, mantienen o amplían el software.

Este tipo de documentación permite comprender cómo está organizado el proyecto y facilita el mantenimiento del sistema cuando se realizan modificaciones o se incorporan nuevas funcionalidades.

Una buena documentación técnica ayuda a:

  • comprender la estructura del proyecto
  • facilitar el mantenimiento del software
  • permitir que otros desarrolladores continúen el trabajo
  • documentar las decisiones tomadas durante el desarrollo

🔶 Contenido de la documentación técnica​

La documentación técnica puede incluir diferentes tipos de información según el proyecto.

Entre los contenidos más habituales se encuentran:

  • Descripción del proyecto. Explica el objetivo de la aplicación y sus funcionalidades principales.
  • Estructura del proyecto. Describe las carpetas, archivos y componentes del sistema.
  • Descripción de módulos o componentes. Explica la función de las partes principales de la aplicación.
  • Dependencias o herramientas utilizadas. Indica las tecnologías o librerías necesarias para ejecutar el proyecto.

Esta información permite que otros desarrolladores comprendan rápidamente cómo está construido el sistema.

Ejemplo fragmento de documentación técnica

Un fragmento de documentación técnica para la aplicación Salón de Héroes podría describir la estructura del proyecto.

salon-heroes/ │ ├── index.html ├── styles/ │ └── main.css ├── js/ │ ├── app.js │ └── heroes.js └── data/ └── heroes.json

  • index.html: página principal de la aplicación.
  • styles/: contiene los archivos de estilo.
  • js/: incluye los scripts que gestionan la lógica de la aplicación.
  • data/: almacena los datos de los héroes.

Esta descripción permite comprender rápidamente cómo está organizado el proyecto.

🛠Herramientas habituales

La documentación técnica puede elaborarse utilizando diferentes herramientas utilizadas habitualmente en proyectos de desarrollo.

Algunas de las más comunes son:

  • Markdown, utilizado para crear documentación sencilla dentro del proyecto.
  • Docusaurus o MkDocs, que permiten generar sitios web de documentación.
  • JSDoc o Dartdoc, que generan documentación automática a partir de comentarios en el código.

Estas herramientas permiten mantener la documentación organizada y facilitar su consulta por parte de los desarrolladores. Actualmente también existen herramientas basadas en inteligencia artificial que pueden ayudar a generar documentación técnica a partir del código del proyecto.

Algunas de las más conocidas son:

  • Mintlify. Plataforma que permite generar documentación automáticamente a partir del código de una aplicación y publicarla en formato web.
  • GitHub Copilot. Asistente de programación basado en inteligencia artificial que puede generar comentarios y explicaciones del código, facilitando la creación de documentación técnica.
  • Sourcegraph Cody. Herramienta que analiza repositorios completos y permite generar explicaciones sobre el funcionamiento del código y sus componentes.
  • Swimm AI. Plataforma que ayuda a crear documentación técnica para desarrolladores conectando la documentación con el código del proyecto.
Uso de herramientas de inteligencia artificial

Las herramientas de inteligencia artificial pueden ayudar a generar documentación de forma más rápida, como manuales de usuario, guías rápidas o documentación técnica.

Sin embargo, el contenido generado debe revisarse siempre para comprobar que la información es correcta, clara y adecuada para los usuarios de la aplicación.
