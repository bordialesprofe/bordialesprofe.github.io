7.1.1 Instalación y verificación del entorno

En esta unidad comenzaremos a construir interfaces con Flutter, por lo que es imprescindible asegurarse de que el entorno de trabajo está correctamente configurado. La instalación completa del SDK de Flutter, Git y VS Code se realizó en la UT5, pero antes de avanzar conviene revisar que todo funciona como debe.

1️⃣ Comprobar la instalación de Flutter​

Ejecuta el siguiente comando en la terminal:

flutter doctor

Este comando analiza el sistema y muestra el estado de cada componente. Para esta unidad deben aparecer correctamente configurados:

  • Flutter
  • Dart
  • VS Code (o el editor utilizado)
  • Chrome (si se trabajará en modo web)

No es necesario corregir advertencias relacionadas con Android Studio o emuladores móviles todavía.

2️⃣ Verificar que el SDK está accesible​

flutter --version

Este comando confirma que Flutter está correctamente instalado y accesible desde cualquier terminal. Si se muestra la versión de Flutter y de Dart, la configuración del PATH es correcta.

3️⃣ Revisar las extensiones esenciales del editor​

En Visual Studio Code deben estar instaladas las siguientes extensiones:

  • Flutter
  • Dart
  • Pubspec Assist (opcional, pero recomendable)

Estas extensiones permiten ejecutar proyectos, autocompletar widgets y gestionar dependencias con facilidad.

4️⃣ Revisar la estructura mínima del proyecto​

Al crear un proyecto con:

flutter create mi_primera_app

Debe generarse una estructura como:

  • lib/ → Código principal de la aplicación
  • web/ → Archivos para ejecutarla en el navegador
  • pubspec.yaml → Archivo de configuración y dependencias

En esta unidad trabajaremos principalmente dentro de lib/.

5️⃣ Ejecutar un proyecto de prueba​

Para comprobar que todo funciona correctamente:

flutter run -d chrome

Si se abre la aplicación por defecto en Chrome, el entorno está preparado para comenzar con los contenidos de la UT7.

Nota importante: errores tras renombrar main.dart

Si el proyecto deja de funcionar después de renombrar archivos (por ejemplo main.dart) o mover ejemplos dentro de lib/, es normal que Flutter muestre errores de pubspec.yaml o de compilación.
Esto ocurre porque Flutter intenta usar información antigua almacenada en sus carpetas de caché.

Para solucionarlo, ejecuta siempre estos pasos:

  1. flutter clean — elimina la carpeta build/ y todos los archivos generados.
  2. flutter pub get — vuelve a generar dependencias y archivos internos.
  3. flutter run — ejecuta el proyecto normalmente.

Estos pasos fuerzan a Flutter a reconstruir el proyecto desde cero y resuelven casi todos los errores provocados por renombrar o mover archivos.
