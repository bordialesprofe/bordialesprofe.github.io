7.4.4 ListView básico

ListView es el widget que permite mostrar contenido en forma de lista desplazable. En esta primera aproximación trabajaremos con una lista estática, es decir, una lista donde los elementos están definidos directamente en el código.

Puedes ampliar esta información

Puedes consultar más ejemplos y variaciones de listas en la documentación oficial de Flutter: https://docs.flutter.dev/ui/widgets/scrolling

1️⃣ ¿Qué es una ListView?​

ListView organiza sus elementos en una columna scrollable. Si el contenido excede el alto de la pantalla, la lista se desplaza automáticamente.

Lista desplazable​

ListView es el widget adecuado cuando se quiere mostrar información en forma de lista vertical.

Comportamiento del scroll en Flutter Web, emuladores y móviles

En una ListView, el scroll solo aparece cuando el contenido es más alto que la pantalla.

En móviles y emuladores, la lista se desplaza con un gesto de arrastre, aunque no aparezca barra de scroll.

En la versión Web, Flutter dibuja toda la interfaz en un canvas (una superficie gráfica única).
Algunos navegadores solo activan el scroll cuando la ventana es más pequeña que ese canvas. Por eso puede parecer que la lista “no se desplaza” aunque el código sea correcto.

Para comprobar el scroll correctamente:

  • usa un dispositivo real o emulador,
  • reduce la altura de la ventana, o
  • incluye suficientes elementos en la lista.

2️⃣ ListView con elementos estáticos (children)​

La forma más simple de usar ListView es mediante la propiedad children, donde añadimos widgets directamente.

  • Ejemplo
  • Resultado
ListView(
  children: const [
    Text('Elemento 1'),
    Text('Elemento 2'),
    Text('Elemento 3'),
    ...
    Text('Elemento 49'),
  ],
)

¿Qué estamos viendo en este ejemplo?

ListView muestra sus elementos uno debajo de otro y permite desplazarse solo cuando el contenido supera la altura de la pantalla.

En Flutter (Android/iOS) no aparece una barra de scroll visible como en las páginas web:
la lista se desplaza con un gesto de arrastre (o arrastrando con el ratón en el emulador).

Ten en cuenta que en la versión Web este comportamiento puede no verse igual:
algunos navegadores no muestran scroll si el canvas (la superficie gráfica donde Flutter Web dibuja toda la aplicación) no cambia de tamaño o si el contenido cabe dentro del área original.

Para comprobar el scroll correctamente, se recomienda:

  • usar un emulador o un móvil real, o
  • reducir realmente la altura de la ventana, o
  • crear una lista lo suficientemente larga.

3️⃣ ListView con ListTile​

ListTile es un widget diseñado para mostrar elementos de lista con una estructura clara y uniforme. Permite incluir iconos, títulos, subtítulos y acciones de forma sencilla, sin tener que construir manualmente una fila (Row) con espaciados.

¿Por qué usar ListTile?

Proporciona un formato listo para usar y es ideal para listas de contactos, menús, ajustes o elementos informativos.

  • Ejemplo
  • Resultado
ListView(
  children: const [
    ListTile(
      leading: Icon(Icons.person),
      title: Text('Usuario 1'),
      subtitle: Text('Activo hace 5 minutos'),
    ),
    ListTile(
      leading: Icon(Icons.person),
      title: Text('Usuario 2'),
      subtitle: Text('En línea'),
    ),
    ListTile(
      leading: Icon(Icons.person),
      title: Text('Usuario 3'),
      subtitle: Text('Desconectado'),
    ),
  ],
)
¿Qué estamos viendo en este ejemplo?

En este ejemplo, cada elemento de la ListView se construye con un ListTile, un widget diseñado para mostrar información de forma organizada.
Cada ListTile incluye:

  • un icono a la izquierda (leading),
  • un título principal (title),
  • y un subtítulo informativo (subtitle).

La lista muestra automáticamente separación entre los elementos y permite scroll si el contenido supera la altura de la pantalla.

4️⃣ ListView dentro de Scaffold (cuando hay más widgets en la pantalla)​

Cuando ListView es el único elemento de la pantalla, puede colocarse directamente en body:. Sin embargo, si necesitamos mostrar otros widgets por encima o por debajo (por ejemplo, un título, una imagen o un botón), debemos usar un Column para organizar la estructura vertical de la pantalla.

problema importante:

ListView no sabe qué altura tiene cuando está dentro de una Column. El resultado es el error típico: Vertical viewport was given unbounded height.

Esto ocurre porque ListView intenta ocupar toda la altura posible, pero Co`lumn no le da un tamaño concreto.

¿Cómo se soluciona? Usando Expanded

Expanded le indica a ListView exactamente cuánto espacio puede ocupar: todo el espacio restante que quede en la Column.

          Column
          │
          ├── Título (altura fija)
          │
          └── Expanded
                │
                └── ListView (ocupa el resto y permite scroll)

Con Expanded, Flutter entiende:"Esta parte de la pantalla puede crecer hasta llenar todo el espacio disponible.”

  • Ejemplo
  • Resultado
Scaffold(
  body: Column(
    children: [
      //Este widget ocupa solo su altura
      Text(
        'Título de la lista',
        style: TextStyle(fontSize: 20),
      ),

      //Este widget dice: "Ocupa el resto del espacio disponible"
      Expanded(
        //Aquí ya podemos poner un ListView sin errores
        child: ListView(
          children: const [
            ListTile(title: Text('Elemento 1')),
            ListTile(title: Text('Elemento 2')),
            ListTile(title: Text('Elemento 3')),
          ],
        ),
      ),
    ],
  ),
);
¿Qué estamos viendo en este ejemplo?

En este ejemplo se muestra un Text fijo en la parte superior y, debajo, una ListView que ocupa todo el espacio restante de la pantalla gracias al uso de Expanded.
El título permanece siempre visible arriba, mientras que los elementos de la lista pueden desplazarse con scroll si hay más contenido del que cabe en la pantalla.

Por qué necesitamos Expanded en este diseño

Para comprender por qué ListView necesita un tamaño definido dentro de una Column, es útil revisar cómo Flutter gestiona los constraints (las reglas que determinan cuánto espacio puede ocupar cada widget).

El equipo de Flutter explica este comportamiento en el siguiente artículo oficial:

🔗 Understanding constraints (Flutter docs) https://docs.flutter.dev/ui/layout/constraints

Este recurso describe por qué widgets como ListView requieren límites concretos y cómo Expanded ayuda a proporcionar ese tamaño dentro de una Column.

Comportamiento del scroll en Flutter Web y emuladores

El scroll en Flutter funciona de forma natural en dispositivos móviles y en emuladores: basta con deslizar con el dedo o arrastrar con el ratón para mover la lista.

En la versión Web este comportamiento puede no verse igual.
Algunos navegadores no muestran barra de desplazamiento y, en ciertos casos, el scroll solo se activa cuando la ventana tiene menos altura que el contenido. Esto puede hacer que parezca que la lista “no se desplaza”, aunque sí funcione en un móvil o en el emulador.

Para comprobar correctamente el scroll:

  • utiliza un móvil real o el emulador de Android, o
  • reduce la altura de la ventana del navegador, o
  • usa una lista con más elementos para forzar el desplazamiento.

Este comportamiento es normal en Flutter Web y no afecta al funcionamiento de las apps móviles.

5️⃣ Actividad: crea tu primera lista estática​

  • Enunciado
  • Código
  • Resultado
  • 🔥 Reto

En esta actividad crearás una pantalla sencilla que muestre una lista estática utilizando ListView y ListTile. El objetivo es practicar la creación de listas básicas en Flutter y comprender cómo organizar elementos verticales con scroll.

Lo que debe incluir tu pantalla

Un título en la parte superior (por ejemplo: “Mis contactos”).

  1. Una ListView con los elementos necesarios para que la pantalla se desplaze.
  2. Cada elemento debe estar formado por un ListTile que incluya:
  • un icono (leading)
  • un texto principal (title)
  • un subtítulo descriptivo (subtitle)

Todo organizado dentro de un widget Column centrado en pantalla.

Resultado esperado

Una pantalla con:

  • Un título fijo en la parte superior
  • Una lista de contactos debajo
  • Elementos con icono + título + subtítulo
  • Scroll si hay más elementos de los que caben
import 'package:flutter/material.dart';

void main() {
  runApp(const MainApp());
}

class MainApp extends StatelessWidget {
  const MainApp({super.key});

  @override
  Widget build(BuildContext context) {
    return const MaterialApp(
      home: HomePage(),
    );
  }
}

class HomePage extends StatelessWidget {
  const HomePage({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Column(
        children: [
          const Padding(
            padding: EdgeInsets.all(16.0),
            child: Text(
              'Mis contactos',
              style: TextStyle(fontSize: 20, fontWeight: FontWeight.bold),
            ),
          ),
          Expanded(
            child: ListView(
              children: const [
                ListTile(
                  leading: Icon(Icons.person),
                  title: Text('Nombre 1'),
                  subtitle: Text('Descripción breve'),
                ),
                // Añade aquí tus ListTile adicionales
              ],
            ),
          ),
        ],
      ),
    );
  }
}
¿Qué estamos viendo en esta actividad?

Cada elemento de la lista se construye con un ListTile, que permite mostrar un icono, un título y un subtítulo de manera ordenada.
El Column organiza la pantalla y Expanded asigna a la lista todo el espacio libre restante, garantizando que el scroll funcione correctamente.

Mejora visual de tu lista

A partir de la lista estática creada en esta actividad, intenta transformarla en una versión más visual y completa.
Para ello, investiga y prueba diferentes widgets y propiedades que ofrece Flutter.

Tu lista debería incluir algunas de estas mejoras:

  • Sustituye el icono de usuario por un CircleAvatar con iniciales o colores personalizados.
  • Añade líneas divisorias entre los elementos empleando Divider.
  • Coloca un icono de acción al final de cada fila utilizando la propiedad trailing.
  • Aplica estilos personalizados a los textos (TextStyle) para diferenciar títulos y subtítulos.
  • Añade Padding o SizedBox para mejorar la separación y la estructura visual.

Tu objetivo es obtener una lista más atractiva, parecida a la de una aplicación real.
No se proporciona el código completo: deberás investigar, probar y combinar los widgets por tu cuenta.

Buenas prácticas al trabajar con ListView
  • Si necesitas mostrar otros widgets encima o debajo de la lista (títulos, botones, buscadores), envuelve la ListView en un Expanded dentro de una Column.
  • Utiliza ListTile para crear elementos de lista claros y consistentes sin necesidad de construir filas manuales con Row.
  • Añade separación visual cuando haya muchos elementos, por ejemplo con Divider o SizedBox(height: ...).
  • Recuerda que el scroll solo aparece cuando el contenido es más alto que la pantalla; con menos de ~10 elementos podría no ser visible.
  • Evita listas muy largas con children:. Para listas extensas o dinámicas, usa ListView.builder (lo veremos más adelante).
  • No añadas ListView dentro de otra ListView sin una razón muy clara (puede provocar problemas de scroll y rendimiento).
