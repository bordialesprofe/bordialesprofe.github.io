7.7.3 setState para interacciones simples

Cuando se pulsa algo y quieres que cambie la interfaz, necesitas:

  1. Un StatefulWidget
  2. Una variable de estado
  3. Llamar a setState() para actualizar la UI

setState() se utiliza para actualizar la interfaz cuando cambia un dato dentro de la misma pantalla.

1️⃣ Ejemplo: Contador con ElevatedButton​

  • Código
  • Visualización
Contador con ElevatedButton
// HomePage ahora es StatefulWidget porque la interfaz va a cambiar (contador).
class HomePage extends StatefulWidget {
// Constructor estándar con key.
const HomePage({super.key});

// OBLIGATORIO en un StatefulWidget:
// Aquí se crea y se asocia el objeto State que guardará el estado del widget.
@override
State<HomePage> createState() => _HomePageState();
}

// Esta clase guarda el estado (variables) y construye la interfaz (build).
class _HomePageState extends State<HomePage> {
// Variable de estado: si cambia y se llama a setState, la UI se actualiza.
int contador = 0;

@override
Widget build(BuildContext context) {
  // Scaffold: estructura básica de pantalla (AppBar + body).
  return Scaffold(
    appBar: AppBar(
      // Título de la barra superior.
      title: const Text('Explorando Flutter'),
    ),
    body: Center(
      // Center centra su child en la pantalla.
      child: Column(
        // Para que la columna ocupe solo el espacio necesario.
        mainAxisSize: MainAxisSize.min,
        children: [
          // Muestra el valor actual del contador en pantalla.
          Text('Contador: $contador'),

          // Separación vertical.
          const SizedBox(height: 12),

          // Botón principal.
          ElevatedButton(
            // Evento del botón: se ejecuta cuando se pulsa.
            onPressed: () {
              // setState avisa a Flutter de que el estado cambió
              // y que debe reconstruir el método build().
              setState(() {
                // Cambio de estado: incrementa el contador en 1.
                contador++;
              });
            },
            // Contenido del botón.
            child: const Text('Sumar'),
          ),
        ],
      ),
    ),
  );
}
}

en este ejemplo
  • Se usa un StatefulWidget porque el valor del contador va a cambiar mientras la app está en ejecución.
  • Se crea una variable de estado llamada contador, que empieza en 0.
  • En el build(), se muestra el valor con Text('Contador: $contador').
  • Al pulsar el botón, se ejecuta onPressed.
  • Dentro de onPressed se llama a setState() para:
    • modificar el estado (contador++)
    • y avisar a Flutter de que debe reconstruir la interfaz.
  • Como build() se vuelve a ejecutar, el Text se actualiza y se ve el nuevo valor.

2️⃣ Ejemplo: Acción en AppBar con IconButton​

  • Código
  • Visualización
Acción en AppBar con IconButton
class HomePage extends StatelessWidget {
  const HomePage({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('Explorando Flutter'),

        // actions: botones a la derecha del AppBar
        actions: [
          IconButton(
            // Icono que se muestra
            icon: const Icon(Icons.info),

            // Texto de ayuda (aparece al mantener pulsado en algunos dispositivos)
            tooltip: 'Mostrar info',

            // Evento: se ejecuta al pulsar el icono
            onPressed: () {
              // Acción simple y visible: SnackBar
              ScaffoldMessenger.of(context).showSnackBar(
                const SnackBar(content: Text('Has pulsado el IconButton')),
              );
            },
          ),
        ],
      ),
      body: const Center(
        child: Text('Pantalla principal'),
      ),
    );
  }
}
en este ejemplo
  • IconButton se coloca en actions del AppBar.
  • Al pulsarlo (onPressed), se ejecuta una acción visible: un SnackBar.
  • No se usa StatefulWidget todavía porque no cambia ningún estado, solo se muestra un aviso.

3️⃣ Ejemplo: Cambio visual con onTap y setState​

  • Código
  • Visualización
Cambiar texto al tocar (mini-setState)
class HomePage extends StatefulWidget {
  const HomePage({super.key});

  @override
  State<HomePage> createState() => _HomePageState();
}

class _HomePageState extends State<HomePage> {
  // Variable de estado que controla el texto mostrado
  String mensaje = 'Toca el texto';

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('Explorando Flutter'),
      ),
      body: Center(
        child: InkWell(
          // Evento que se ejecuta al tocar el texto
          onTap: () {
            // setState indica que el estado ha cambiado
            setState(() {
              // Cambio simple del estado
              mensaje = 'Texto pulsado';
            });
          },
          child: Text(mensaje),
        ),
      ),
    );
  }
}
en este ejemplo
  • Se usa StatefulWidget porque el texto puede cambiar.
  • mensaje es la variable de estado.
  • onTap detecta el toque sobre el texto.
  • setState() actualiza el valor y Flutter reconstruye la interfaz.
  • El cambio es inmediato y visible.
