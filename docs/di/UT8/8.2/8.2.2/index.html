8.2.2 Separar los datos y el modelo

Hasta ahora la pantalla contiene demasiada información: además de la interfaz y la lógica, también incluye todos los datos de los héroes. En este apartado se separan dos cosas:

  • Modelo: una clase para representar un héroe (HeroeModel)
  • Datos: la lista de héroes en un archivo independiente

Así la pantalla se limita a mostrar y reaccionar, sin “almacenar” toda la información.

A continuación se detallan los pasos para separar los datos.

1️⃣ Crear el modelo (models/)​

Se define una clase para representar un héroe con sus campos básicos.

  • Crear lib/models/heroes_model.dart
  • Definir la clase con sus propiedades (nombre, clase, descripcion, imagePath)

2️⃣ Crear el archivo de datos (data/)​

Se mueve la lista de héroes fuera de la pantalla para no mezclar UI con datos.

  • Crear lib/data/heroes_data.dart
  • Declarar la lista heroes usando el modelo

3️⃣ Actualizar la pantalla para usar los datos​

La pantalla recorre la lista heroes y genera las tarjetas dinámicamente.

  • Importar heroes_data.dart
  • Sustituir ListView(children: ...) por ListView.builder o ListView.separated
  • Generar cada tarjeta desde itemBuilder recorriendo heroes por índice
  • Mantener la llamada a heroCard(...) sin cambios
  • Si se usa ListView.separated, definir el espaciado entre tarjetas en separatorBuilder
  • Ejecutar la app y comprobar que la funcionalidad es la misma
ListView.builder o ListView.separated

Aunque la aplicación sigue funcionando correctamente con ListView(children: ...), cuando los datos se importan desde un archivo externo se recomienda usar ListView.builder o ListView.separated porque son el patrón habitual para listas dinámicas y escalables.

  • ListView(children: ...) crea toda la lista de golpe Flutter construye todos los widgets desde el inicio, aunque no se vean en pantalla

  • ListView.builder crea solo lo necesario Construye los elementos bajo demanda según se van mostrando al hacer scroll. Es más eficiente cuando la lista crece

  • ListView.separated hace lo mismo y además separa el espaciado Permite definir el SizedBox en un separatorBuilder, sin meterlo dentro del listado. El código queda más limpio

  • Es el patrón más habitual en proyectos reales Cuando los datos vienen de un archivo externo, una API o una base de datos, lo normal es usar builder o separated

ListView.separated no “recorre” la lista como un for literal, construye automáticamente los elementos llamando a itemBuilder con index desde 0 hasta itemCount - 1.

// lib/screens/salon_heroes_screen.dart
import '../data/heroes_data.dart';
import '../models/heroes_model.dart';

class SalonHeroesScreen extends StatefulWidget {
const SalonHeroesScreen({super.key});

@override
State<SalonHeroesScreen> createState() => \_SalonHeroesScreenState();
}

class \_SalonHeroesScreenState extends State<SalonHeroesScreen> {
String heroeSeleccionado = '';

@override
Widget build(BuildContext context) {
return Scaffold(
appBar: AppBar(
title: const Text(
'Salón de héroes',
style: TextStyle(fontWeight: FontWeight.bold),
),
centerTitle: true,
backgroundColor: const Color(0xC1035B02),
foregroundColor: const Color(0xC1FFFFFF),
actions: [
IconButton(
icon: const Icon(Icons.refresh),
tooltip: 'Limpiar selección',
onPressed: () {
setState(() {
heroeSeleccionado = '';
});
},
),
],
),
body: Padding(
padding: const EdgeInsets.all(16),
child: ListView.separated(
// Numero total de elementos que se van a pintar en la lista
itemCount: heroes.length,

          // Widget separador entre elementos
          // Se ejecuta entre cada item y el siguiente (no al final)
          separatorBuilder: (context, index) => const SizedBox(height: 12),

          // Constructor de cada elemento de la lista
          // Flutter llama a este builder solo para los items que necesita mostrar
          itemBuilder: (context, index) {
            // Se obtiene el heroe correspondiente a la posicion actual
            final HeroeModel heroe = heroes[index];

            // Se devuelve la tarjeta del heroe usando el componente existente
            return heroCard(
              context: context,
              imagePath: heroe.imagePath,
              nombre: heroe.nombre,
              clase: heroe.clase,
              descripcion: heroe.descripcion,
              seleccionado: heroeSeleccionado == heroe.nombre,
            );
          },
        ),
      ),
    );

}

Widget heroCard({
required BuildContext context,
required String imagePath,
required String nombre,
required String clase,
required String descripcion,
required bool seleccionado,
}) {
return InkWell(
onTap: () {
setState(() {
heroeSeleccionado = nombre;
});

        ScaffoldMessenger.of(
          context,
        ).showSnackBar(SnackBar(content: Text(descripcion)));
      },
      child: Container(
        padding: const EdgeInsets.all(12),
        decoration: BoxDecoration(
          color: seleccionado ? const Color(0xFFDFF2DF) : Colors.white,
          borderRadius: BorderRadius.circular(12),
          border: Border.all(
            color: const Color(0xC1035B02),
            width: seleccionado ? 4 : 2,
          ),
        ),
        child: Row(
          children: [
            Image.asset(imagePath, width: 64, height: 64),
            const SizedBox(width: 12),
            Column(
              crossAxisAlignment: CrossAxisAlignment.start,
              children: [
                Text(
                  nombre,
                  style: const TextStyle(
                    fontSize: 18,
                    fontWeight: FontWeight.bold,
                  ),
                ),
                Text(
                  clase,
                  style: const TextStyle(fontSize: 14, color: Colors.grey),
                ),
              ],
            ),
          ],
        ),
      ),
    );

}
}
