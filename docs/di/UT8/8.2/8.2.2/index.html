8.2.2 Modelos y datos

Antes de trabajar con pantallas o componentes visuales, es importante definir correctamente qué datos maneja la aplicación y cómo se representan. Para ello se separan dos conceptos distintos: el modelo y los datos.

El modelo define la estructura de la información (qué campos tiene un elemento y de qué tipo son), mientras que los datos representan el contenido concreto que usa la aplicación (listas, valores iniciales, ejemplos).

Separar estos dos elementos permite que la aplicación sea más clara, más fácil de mantener y más sencilla de ampliar en el futuro.

Idea clave

El modelo define la forma de los datos y los datos definen el contenido. Cada uno debe tener su propio lugar.

1️⃣ Modelo de datos​

El modelo se coloca en la carpeta models/ y se representa mediante clases o enumeraciones. Estas clases no contienen lógica de interfaz ni código visual, solo describen la información que maneja la aplicación.

En una app como Salón de Héroes, el modelo principal puede ser un héroe, con propiedades como su nombre, tipo o imagen. Además, ciertos valores cerrados, como el tipo de héroe, se pueden representar de forma clara mediante un enum.

Nombre de archivos y clases

El modelo se define en un archivo independiente, por ejemplo hero_model.dart, y se representa mediante una clase con el mismo nombre en formato UpperCamelCase.

Modelo principal
class HeroModel {
  final String nombre;
  final HeroType tipo;

  const HeroModel({
    required this.nombre,
    required this.tipo,
  });
}
enum para los tipos de héroe
enum HeroType {
  guerrero,
  mago,
  arquero,
  sanador,
}

2️⃣ Datos de la aplicación​

Los datos de la aplicación se colocan en la carpeta data/ y representan el contenido concreto que la app utiliza. A diferencia del modelo, aquí no se define la estructura, sino los valores reales con los que trabaja la aplicación.

Estos datos suelen definirse en listas o constantes y utilizan los modelos creados previamente. De este modo, se garantiza que toda la información tenga una estructura coherente y bien definida.

En una app como Salón de Héroes, los datos pueden ser una lista inicial de héroes que se mostrará en la pantalla principal.

Nombre de archivos y clases

Los datos se almacenan en un archivo específico, por ejemplo heroes_data.dart, que contiene listas o constantes basadas en los modelos definidos.

Datos iniciales
final List<HeroModel> heroes = [
  HeroModel(
    nombre: 'Aldric',
    tipo: HeroType.guerrero,
  ),
  HeroModel(
    nombre: 'Lyra',
    tipo: HeroType.mago,
  ),
  HeroModel(
    nombre: 'Kael',
    tipo: HeroType.arquero,
  ),
];

Estos datos se almacenan fuera de las pantallas y se importan solo cuando son necesarios. De este modo, la interfaz no contiene información “hardcodeada” y el código resulta más limpio y fácil de mantener.
