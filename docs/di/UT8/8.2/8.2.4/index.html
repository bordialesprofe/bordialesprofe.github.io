8.2.4 Estilos compartidos

Cuando una aplicación crece, es habitual que se repitan estilos en distintos widgets y pantallas (colores, tamaños de texto, bordes, rellenos o sombras). Para evitar duplicación y mantener una estética coherente, estos valores se centralizan como estilos compartidos.

En Flutter, una parte importante del estilo puede definirse de forma global mediante el tema de la aplicación. En proyectos sencillos, conviene agruparlo en un único archivo para tenerlo todo localizado y facilitar cambios rápidos.

Idea clave

Centralizar estilos no cambia la interfaz de la app, pero hace que sea más consistente y más fácil de mantener.

1️⃣ Dónde se definen los estilos​

Los estilos compartidos se colocan en la carpeta styles/. En una app sencilla como Salón de Héroes, se puede utilizar un único archivo de tema para agrupar lo principal.

Nombre de archivos y estilos

El estilo global se centraliza en un archivo como app_theme.dart. Este archivo define el tema de la aplicación y se aplica desde main.dart.

2️⃣ Tema global (ejemplo)​

En app_theme.dart se define el tema general de la aplicación. Aquí se suelen incluir colores y estilos de texto base para que toda la app mantenga una estética uniforme.

Ejemplo tema global
class AppTheme {
  static ThemeData theme() {
    final colorScheme = ColorScheme.fromSeed(seedColor: Colors.deepPurple);

    return ThemeData(
      useMaterial3: true,
      colorScheme: colorScheme,
      textTheme: const TextTheme(
        titleLarge: TextStyle(fontSize: 22, fontWeight: FontWeight.w700),
        bodyMedium: TextStyle(fontSize: 16),
      ),
    );
  }
}

Después, el tema se aplica en main.dart dentro de MaterialApp.

en este ejemplo

En este ejemplo se utiliza ColorScheme.fromSeed(...) para generar automáticamente una paleta completa de colores a partir de un único color base (seedColor). Flutter calcula a partir de esa “semilla” los colores principales y secundarios que necesita el tema (por ejemplo, primary, secondary, surface o onPrimary), manteniendo una estética coherente sin tener que definir cada color manualmente. Esta opción es muy útil en proyectos sencillos o en prácticas, porque permite cambiar el estilo general de la app modificando solo el seedColor.
