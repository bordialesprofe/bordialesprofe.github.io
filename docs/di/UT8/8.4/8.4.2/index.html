8.4.2 Detectar tamaño de pantalla con MediaQuery

Para aplicar diseño adaptativo o responsive, lo primero es conocer el espacio disponible.
En Flutter, MediaQuery permite obtener información del dispositivo y del tamaño de pantalla.

En Salón de Héroes, esto permite decidir si conviene mostrar solo la lista (móvil) o aprovechar el ancho para mostrar lista y detalle (pantalla grande).

1️⃣ Obtener ancho y alto de pantalla​

Con MediaQuery.of(context).size se obtiene el tamaño de la pantalla.

final size = MediaQuery.of(context).size;
final ancho = size.width;
final alto = size.height;
  • width es el ancho disponible en píxeles lógicos.
  • height es el alto disponible en píxeles lógicos.

2️⃣ Usar un breakpoint (umbral) para decidir el layout​

Un breakpoint es un valor de referencia (normalmente un ancho) a partir del cual se cambia la estructura del diseño.

En lugar de verlo como un “número mágico”, se elige cuando el contenido deja de verse cómodo.

Por ejemplo:

  • móvil: ancho menor de 600
  • pantalla grande: ancho mayor o igual a 600

Nota: el valor exacto del breakpoint no es obligatorio. Se elige según el tipo de interfaz y el contenido de la app.

@override
Widget build(BuildContext context) {
  final ancho = MediaQuery.of(context).size.width;
  // Breakpoint: a partir de este ancho cambiamos el layout
  if (ancho < 600) {
    return _buildLayoutMovil();
  } else {
    return _buildLayoutPantallaGrande();
  }
}

La aplicación sigue siendo la misma. Lo que cambia es la organización de los widgets.

3️⃣ ¿Cuándo usar MediaQuery?​

Se recomienda usar MediaQuery cuando:

  • Se quiere tomar una decisión global sobre el diseño.
  • Se necesita conocer el tamaño total de la pantalla.
  • Se quiere adaptar márgenes o espaciados generales.

Si se necesita conocer el espacio disponible dentro de un widget concreto, es preferible usar LayoutBuilder, que se estudiará en el siguiente apartado.

4️⃣ Ejemplo aplicado a Salón de Héroes​

En este ejemplo, la pantalla muestra:

  • Móvil (ancho < 600): solo la lista de héroes.
  • Pantalla grande (ancho >= 600): lista a la izquierda y detalle a la derecha.
  • main.dart
  • screens/salon_heroes_screen.dart
import 'package:flutter/material.dart';
import 'screens/salon_heroes_screen.dart';
void main() {
runApp(const MyApp());
}
class MyApp extends StatelessWidget {
const MyApp({super.key});
@override
Widget build(BuildContext context) {
  return MaterialApp(
    debugShowCheckedModeBanner: false,
    title: 'Salón de Héroes',
    home: const SalonHeroesScreen(),
  );
}
}
import 'package:flutter/material.dart';

class SalonHeroesScreen extends StatefulWidget {
const SalonHeroesScreen({super.key});

@override
State<SalonHeroesScreen> createState() => _SalonHeroesScreenState();
}

class _SalonHeroesScreenState extends State<SalonHeroesScreen> {
final heroes = const ['Arthos', 'Lyra', 'Dorian', 'Morgana'];
String? heroeSeleccionado;

@override
Widget build(BuildContext context) {
// DISEÑO ADAPTATIVO: se obtiene el ancho de pantalla con MediaQuery
final ancho = MediaQuery.of(context).size.width;

return Scaffold(
  appBar: AppBar(title: const Text('Salón de Héroes')),

  // DISEÑO ADAPTATIVO: se decide qué layout mostrar según el ancho
  // - Móvil: ancho < 600 → solo lista
  // - Pantalla grande: ancho >= 600 → lista + detalle
  body: ancho < 600 ? _buildMovil() : _buildPantallaGrande(),
);
}

Widget _buildMovil() {
return ListView.builder(
  itemCount: heroes.length,
  itemBuilder: (context, index) {
    final heroe = heroes[index];
    return ListTile(
      title: Text(heroe),
      onTap: () {
        // En móvil lo habitual sería navegar a una pantalla de detalle.
        // Aquí se deja un comportamiento mínimo para el ejemplo.
        ScaffoldMessenger.of(context).showSnackBar(
          SnackBar(content: Text('Abrir detalle de $heroe')),
        );
      },
    );
  },
);
}

Widget _buildPantallaGrande() {
// DISEÑO ADAPTATIVO: en pantallas grandes se aprovecha el espacio
// mostrando lista y detalle a la vez en una fila (Row)
return Row(
  children: [
    // DISEÑO ADAPTATIVO: Expanded reparte el espacio disponible
    // para que ambos paneles se ajusten a cualquier ancho
    Expanded(child: _buildListaEscritorio()),

    // DISEÑO ADAPTATIVO: segundo panel flexible para el detalle
    Expanded(child: _buildDetalle()),
  ],
);
}

Widget _buildListaEscritorio() {
return ListView.builder(
  itemCount: heroes.length,
  itemBuilder: (context, index) {
    final heroe = heroes[index];
    final seleccionado = heroe == heroeSeleccionado;

    return ListTile(
      selected: seleccionado,
      title: Text(heroe),
      onTap: () {
        // En pantallas grandes no se navega: se actualiza el panel de detalle
        setState(() {
          heroeSeleccionado = heroe;
        });
      },
    );
  },
);
}

Widget _buildDetalle() {
if (heroeSeleccionado == null) {
  return const Center(
    child: Text('Selecciona un héroe para ver el detalle'),
  );
}

return Padding(
  padding: const EdgeInsets.all(16),
  child: Text(
    'Detalle de $heroeSeleccionado\n\nAquí se mostrarían sus datos.',
  ),
);
}
}
