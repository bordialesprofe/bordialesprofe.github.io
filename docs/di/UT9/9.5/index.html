9.5 Distribución Web

Una vez que la aplicación funciona correctamente, se puede generar la versión Web en modo release y publicarla en un canal accesible desde cualquier dispositivo.

En esta unidad se utilizará Vercel como plataforma de despliegue, ya que permite publicar una web estática de forma rápida y obtener una URL pública.

Vercel es una plataforma de despliegue en la nube que permite publicar aplicaciones web de forma rápida y sencilla. Está especialmente orientada a proyectos frontend y sitios estáticos, y ofrece un proceso de publicación automatizado mediante línea de comandos o integración con repositorios Git.

En el contexto de esta unidad, Vercel se utilizará para:

  • Publicar la versión web generada con flutter build web.
  • Obtener una URL pública accesible desde cualquier dispositivo.
  • Distribuir la aplicación como sitio web o instalarla como PWA.

1️⃣ Generar build web en modo release​

🔶 Paso 1 Construir la versión web optimizada

flutter build web --release

Este comando:

  • Compila la aplicación para Web.
  • Genera una versión optimizada (release).
  • Crea los archivos listos para desplegar.

🔶 Paso 2 Localizar la carpeta generada

Ruta habitual:

build/web

2️⃣ Despliegue en Vercel con CLI​

Para publicar el contenido de build/web, se utiliza la herramienta de línea de comandos de Vercel.

🔶 Paso 1 Instalar la CLI de Vercel

npm i -g vercel

🔶 Paso 2 Acceder a la carpeta del build

cd build/web

🔶 Paso 3 Desplegar el proyecto

vercel

Durante el asistente, se seleccionan opciones como:

  • Scope o cuenta donde se creará el proyecto.
  • No vincular a un proyecto existente.
  • Nombre recomendado: salon-heroes
  • Framework: Other
  • Directorio del código: .

Al finalizar se obtiene una URL temporal de preview para comprobar que todo funciona correctamente (carga, navegación, estilos, etc.).

en este ejemplo

Al ejecutar vercel, pueden aparecer varias URLs en la terminal. Es importante distinguirlas:

  • Inspect:
    URL interna del panel de Vercel.
    Sirve para gestionar el proyecto desde la web de Vercel, pero no es la URL pública para compartir.

  • Production (URL larga con identificador):
    Ejemplo:
    https://salon-heroes-mwjn3l6z-bordialesrprofe-6196s-projects.vercel.app
    Es la URL específica de ese despliegue concreto.
    Incluye un identificador único y puede cambiar en cada publicación.

  • Aliased (URL corta y limpia):
    Ejemplo:
    https://salon-heroes-web.vercel.app
    Es la URL principal y estable del proyecto.
    Es la que normalmente se comparte.

📌 En la práctica, se recomienda compartir la URL Aliased, ya que es la definitiva del proyecto.

Autenticación Vercel

Si es la primera vez que se utiliza la CLI de Vercel en el equipo, al ejecutar vercel será necesario iniciar sesión. La terminal mostrará un enlace de autenticación. Se debe:

  1. Abrir el enlace en el navegador.
  2. Iniciar sesión con la cuenta deseada (por ejemplo, Google).
  3. Confirmar la autorización del dispositivo. Una vez completado el proceso, la CLI quedará vinculada a esa cuenta y mostrará el scope disponible para crear proyectos.

🔶 Paso 4 Publicar en producción

Cuando ya se ha comprobado que la preview funciona, se publica la versión final en producción:

vercel --prod

Este comando:

  • Publica la aplicación como versión oficial en producción.
  • Sobrescribe la versión anterior si ya existía.
  • Mantiene la URL principal del proyecto (por ejemplo, https://salon-heroes.vercel.app).
404 al recargar rutas internas (Flutter Web / SPA)

En Flutter Web, la aplicación suele comportarse como una SPA (Single Page Application).
Esto significa que el navegador carga una sola página real (index.html) y Flutter muestra el resto de pantallas internamente.

🔹 Ejemplo (Detalle héroe)

  • Home: /
  • Detalle del héroe 1: /detalle/1

✅ Si se navega desde dentro de la app de / a /detalle/1, normalmente funciona.
❌ Pero si estando en /detalle/1 se recarga la página (F5) o se entra directamente por esa URL, Vercel intenta buscar un archivo real llamado /detalle/1.
Como ese archivo no existe (solo existe index.html), aparece un 404.

🔹 Solución en Vercel (rewrites)

En la carpeta que se despliega (build/web), crear un archivo vercel.json con este contenido:

{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
´´´
Después, volver a desplegar la aplicación en Vercel para aplicar el cambio.

📌 Esta configuración solo es necesaria si la aplicación utiliza rutas visibles en la URL.

3️⃣ Instalar como PWA en Chrome​

  1. Abrir la URL de Vercel en Chrome.
  2. Usar:

• Botón Instalar en la barra, o

• Menú (⋮) → Instalar aplicación

¿Qué se consigue al instalar como PWA?

La web se comporta como una app. Se abre en una ventana propia, sin pestañas del navegador.

4️⃣ Distribución mediante código QR​

Una vez que la aplicación web está publicada en Vercel y dispone de una URL pública, puede distribuirse fácilmente mediante un código QR.

🔶 Paso 1 Copiar la URL pública

Ejemplo: https://salon-heroes-web.vercel.app

🔶 Paso 2 Generar el código QR

Se pueden utilizar herramientas gratuitas como:

  • The QR Code Generator → https://www.the-qrcode-generator.com/
  • QR Code Generator (goqr.me) → https://goqr.me/
  • Canva (herramienta integrada de QR dentro del editor)

👉 Antes de compartir el QR comprueba:

      1. Que la URL funciona correctamente
      2. Que la versión desplegada es la última
      3. Que el QR se escanea correctamente
