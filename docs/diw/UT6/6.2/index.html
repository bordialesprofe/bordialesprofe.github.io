6.2 Bases técnicas del diseño responsive

Antes de adaptar un diseño a distintos dispositivos, conviene preparar una base “flexible” para evitar problemas típicos como desbordes, contenidos que se salen del contenedor o imágenes que rompen el layout. Estas bases son el punto de partida habitual en cualquier diseño responsive.

1️⃣ Configuración del viewport​

El viewport es el área visible donde el navegador “dibuja” la página. En diseño responsive es importante porque el navegador calcula tamaños y distribuciones (porcentajes, vw, columnas, etc.) en función de ese viewport.

En móviles, si no se indica nada, el navegador puede usar un viewport “tipo escritorio” y escalar la web para que quepa. Esto provoca efectos típicos: texto demasiado pequeño, necesidad de hacer zoom y layouts que no se adaptan como esperas.

Para evitarlo:

Añadir en el head
<meta name="viewport" content="width=device-width, initial-scale=1">
  • width=device-width: usa el ancho real del dispositivo como ancho del viewport.
  • initial-scale=1: carga la página a escala 1 (sin zoom inicial).

🔶 Valores habituales en content

  • width: define el ancho del viewport. Lo más común es width=device-width (ancho real del dispositivo). También se puede fijar un número (width=500), aunque en responsive no suele ser buena idea.
  • initial-scale: indica la escala inicial al cargar la página. Lo habitual es initial-scale=1.
  • minimum-scale y maximum-scale: establecen el zoom mínimo y máximo permitido.
  • user-scalable: permite o impide que el usuario haga zoom (yes o no). Por accesibilidad, es preferible no bloquear el zoom.

Con esta configuración, el diseño se comporta de forma más predecible y la web puede adaptarse correctamente a pantallas pequeñas.

Accesibilidad y zoom

Evita usar user-scalable=no o limitar el zoom con maximum-scale (por ejemplo, maximum-scale=1), porque puede crear problemas de accesibilidad. En la mayoría de casos basta con width=device-width, initial-scale=1.

2️⃣ box-sizing: border-box​

Cuando se trabaja con diseños responsive es muy habitual definir anchos flexibles (por ejemplo, width: 100% o porcentajes) y añadir padding para separar el contenido. El problema es que, por defecto, el navegador usa box-sizing: content-box, lo que significa:

  • el width solo mide el contenido
  • el padding y el border se suman aparte

Resultado: el elemento puede terminar midiendo más de lo que esperas y provocar desbordes o barras de scroll horizontales.

tip

Si algo “se sale” del contenedor en móvil y no sabes por qué, revisa primero padding y box-sizing.

Con box-sizing: border-box, el ancho que indicas incluye también el padding y el borde, por lo que el tamaño final es más predecible y fácil de mantener en layouts flexibles.

recomendación
box-sizing reset
*, *::before, *::after {
  box-sizing: border-box;
}

Aplica box-sizing: border-box a todos los elementos y también a ::before y ::after, de forma que el ancho y el alto incluyen el padding y el borde. Así se evitan desbordes y el tamaño de las cajas es más predecible.

3️⃣ Contenedores fluidos con límites​

En responsive, lo habitual es que los contenedores principales sean fluidos: ocupan todo el ancho disponible en pantallas pequeñas para aprovechar el espacio. Sin embargo, en pantallas grandes conviene establecer un límite máximo para que el contenido no quede demasiado “estirado” y sea más cómodo de leer.

Patrón típico
/* Combinar width: 100% con max-width y centrar el contenedor */ 
.contenedor {
  width: 100%;
  max-width: 1200px;
  margin: 0 auto;
  padding: 16px;
}
  • width: 100% hace que el contenedor se adapte al ancho disponible.
  • max-width evita que se expanda sin control en pantallas grandes.
  • margin: 0 auto lo centra cuando hay espacio sobrante.
  • padding añade aire interior para que el contenido no “toque” los bordes.
tip

Si el texto es largo, poner un max-width mejora mucho la legibilidad en monitores grandes.

4️⃣ Imágenes y vídeos fluidos​

En responsive, uno de los problemas más comunes es que una imagen o un vídeo tenga un tamaño mayor que su contenedor y rompa el diseño (aparece scroll horizontal o se descuadra el layout). Por eso conviene hacer que el contenido multimedia sea flexible.

Regla básica
img, video {
  max-width: 100%;
  height: auto;
}
  • max-width: 100% evita que la imagen o el vídeo sean más anchos que su contenedor.
  • height: auto mantiene la proporción para que no se deformen al reducirse.
tip

Si una página “se sale” por los lados en móvil, revisa primero imágenes, vídeos o iframes con tamaños fijos.

5️⃣ Evitar anchos rígidos en elementos principales​

Para que un diseño sea realmente responsive, conviene evitar anchos fijos grandes en px (por ejemplo, contenedores a 900px o tarjetas a 400px). En pantallas pequeñas esos valores no caben y el layout se rompe.

En su lugar, se suele trabajar con:

  • porcentajes (width: 100%, width: 90%)
  • límites (max-width, min-width)
  • layouts flexibles (Flexbox o Grid)
Porcentajes y unidades viewport

Los porcentajes no dependen del tamaño de la ventana, dependen del tamaño del contenedor. En width suelen funcionar sin problemas, pero en height solo funcionan si el elemento padre tiene una altura definida. Si necesitas basarte en el tamaño de la pantalla, usa unidades de viewport (vw, vh).

Ejemplo
.card {
  width: 100%;
  max-width: 420px;
}
tip

Reserva px para detalles que necesitan precisión (bordes, iconos, separaciones pequeñas) y usa unidades flexibles para la estructura del layout.
