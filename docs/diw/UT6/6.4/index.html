6.4 Media features

Además del ancho de pantalla, las media queries pueden comprobar otras características del dispositivo o del entorno (orientación, proporción, tipo de puntero, etc.). Estas “características” se llaman media features y se escriben dentro de los paréntesis de @media.

Más información

En este apartado se recogen las media features más utilizadas en responsive. Si necesitas consultar el listado completo, puedes verlo en MDN Web Docs (Media features) o en LenguajeCSS (Media Features).

1️⃣ Tamaño de pantalla​

Se usa para ajustar el diseño según el ancho o el alto disponible.

  • Ejemplo ancho
  • Ejemplo alto
@media (width <= 600px) {
  .menu { font-size: 14px; }
}
en este ejemplo

Cuando el ancho es 600px o menos, el menú reduce el tamaño de letra para mejorar la legibilidad.

@media (height <= 500px) {
  .hero { display: none; }
}
en este ejemplo

Si el alto es pequeño (por ejemplo, móvil girado en horizontal), se oculta un bloque grande para ganar espacio.

2️⃣ Orientación del dispositivo​

Permite detectar si la pantalla está en vertical o en horizontal.

  • Ejemplo vertical
  • Ejemplo horizontal
@media (orientation: portrait) {
  .galeria { grid-template-columns: 1fr; }
}
en este ejemplo

En vertical la galería se muestra en una sola columna porque suele haber menos ancho.

    @media (orientation: landscape) {
      .galeria { grid-template-columns: repeat(3, 1fr); }
    }
en este ejemplo

En horizontal se aprovecha el ancho extra para mostrar más columnas.

3️⃣ Proporción de pantalla​

A veces no importa solo el tamaño, sino la forma de la pantalla (más panorámica o más “cuadrada”). Para eso se usa la proporción.

  • Ejemplo pantalla panorámica
  • Ejemplo pantalla es más alta que ancha
  @media (aspect-ratio >= 16/9) {
    .video { max-width: 900px; margin: 0 auto; }
  }
en este ejemplo

Si la pantalla es panorámica, se limita el ancho del vídeo para que no quede excesivamente estirado y se centra.

  @media (aspect-ratio < 1/1) {
    .paneles { flex-direction: column; }
  }
en este ejemplo

Si la pantalla es más alta que ancha (típico móvil en vertical), los paneles se apilan en columna.

4️⃣ Tipo de interacción​

Estas features ayudan a adaptar el diseño según si el usuario usa ratón o táctil.

  • Ejemplo pantalla panorámica
  • Ejemplo pantalla es más alta que ancha
  @media (hover: hover) {
    .card:hover { transform: translateY(-4px); }
  }
en este ejemplo

El efecto hover solo se aplica cuando el dispositivo realmente soporta “pasar por encima” (ratón/trackpad).

  @media (pointer: coarse) {
    .boton { padding: 16px 18px; }
  }
en este ejemplo

Si el puntero es “grueso” (dedo), se agrandan las zonas clicables para evitar fallos al pulsar.

5️⃣ Combinar condiciones​

Puedes combinar varias condiciones con and para afinar casos concretos.

  • Ejemplo pantalla estrecha y en horizontal
    @media (width <= 700px) and (orientation: landscape) {
      .navbar { position: static; }
    }
en este ejemplo

El ajuste solo ocurre en pantallas estrechas y en horizontal (caso típico: móvil girado).
