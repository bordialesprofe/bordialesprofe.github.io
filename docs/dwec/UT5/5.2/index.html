5.2 Definición e invocación de funciones

Para poder utilizar una función en un programa, es necesario definirla previamente. Definir una función consiste en indicar qué hace y cómo se llama, pero no implica que se ejecute.

Una función se define una sola vez, pero puede invocarse tantas veces como sea necesario a lo largo del programa.

1️⃣ Definición de una función​

La definición de una función indica:

  • el nombre de la función,
  • las instrucciones que se ejecutarán cuando se llame.
function saludar() {
  console.log("Hola desde la función");
}

En este ejemplo:

  • saludar es el nombre de la función.
  • El código entre { } es el cuerpo de la función.
  • La función queda preparada para ser utilizada, pero aún no se ejecuta.
nota

Definir una función no provoca ninguna salida ni efecto visible si no se invoca.

2️⃣ Invocación de una función​

Invocar una función (también llamado llamar a la función) consiste en escribir su nombre seguido de paréntesis.

saludar();

Cuando el programa llega a esa línea:

  1. Busca la definición de la función.
  2. Ejecuta, una a una, las instrucciones de su interior.
  3. Finaliza la función y vuelve al punto desde el que fue llamada.
info

Aunque una función no reciba parámetros, los paréntesis son obligatorios al invocarla.

3️⃣ Flujo de ejecución​

Cuando una función es invocada:

  • el programa interrumpe momentáneamente su ejecución normal,
  • ejecuta el código de la función,
  • y después continúa justo donde se había quedado.

Este comportamiento permite estructurar el programa en bloques claros y reutilizables.

tip

Es habitual definir las funciones al inicio del archivo y realizar las invocaciones en el programa principal.
