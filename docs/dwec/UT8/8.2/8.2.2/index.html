8.2.2 Tipos de eventos

Los eventos se pueden clasificar según la acción que los provoca. En esta unidad se trabajarán los más habituales en aplicaciones web interactivas: eventos del ratón, del teclado, de formularios, de carga y algunos otros muy usados.

1️⃣ Eventos del ratón (mouse)​

Los eventos del ratón se disparan cuando el usuario interactúa con elementos usando el puntero: clics, doble clic, pasar por encima o sacar el ratón de un elemento. Son los más frecuentes en botones, tarjetas y menús.

EventoDescripciónEjemplo de uso
clickClic sobre un elementoBotón de “Enviar”, “Eliminar”, “Abrir modal”
dblclickDoble clicEditar un elemento rápidamente
mouseenterEl ratón entra en un elementoMostrar una ayuda o resaltar una tarjeta
mouseleaveEl ratón sale de un elementoOcultar una ayuda o quitar el resaltado
  • HTML
  • JavaScript
<button id="btn-mouse">Pulsar</button> 

boton.addEventListener("click", () => {
boton.classList.toggle("activo");
});
en este ejemplo

Al hacer clic en el botón, se alterna la clase activo para cambiar su estado visual.

2️⃣ Eventos del teclado (keyboard)​

Los eventos de teclado se producen cuando el usuario escribe o pulsa teclas. Se usan para atajos, control por teclado y validaciones mientras se escribe en campos de texto.

EventoDescripciónEjemplo de uso
keydownSe pulsa una teclaDetectar atajos o teclas especiales
keyupSe suelta una teclaValidar al terminar de pulsar
  • HTML
  • JavaScript
<input id="entrada" type="text" placeholder="Escribe algo..." /> 
const entrada = document.querySelector("#entrada");

entrada.addEventListener("keydown", () => {
entrada.classList.add("escribiendo");
});
en este ejemplo

Cuando se pulsa una tecla dentro del input, se añade una clase para reflejar que el usuario está escribiendo.

3️⃣ Eventos de formularios​

Los eventos de formularios permiten reaccionar cuando el usuario modifica un campo, envía un formulario o cambia el valor de un control. Son imprescindibles para validación y para evitar envíos incorrectos.

EventoDescripciónEjemplo de uso
inputEl valor cambia mientras se escribeValidación en tiempo real
changeEl valor cambia y se confirmaSelects, checkboxes o inputs al perder foco
submitSe envía un formularioCapturar envío y usar preventDefault()
focusEl campo recibe el focoMarcar visualmente el campo activo
blurEl campo pierde el focoValidar al salir de un campo
  • HTML
  • JavaScript
<form id="form-aviso"> 
  <input id="titulo" type="text" placeholder="Título" /> 
  <button type="submit">Guardar</button> 
</form> 
const form = document.querySelector("#form-aviso");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  alert("Formulario capturado sin recargar la página");
});
en este ejemplo

Se captura el envío del formulario y se evita la recarga usando preventDefault().

4️⃣ Eventos de carga​

Estos eventos se disparan cuando la página o un recurso termina de cargarse. Se usan para ejecutar código cuando el DOM ya está disponible y se pueden seleccionar elementos sin errores.

EventoDescripciónEjemplo de uso
DOMContentLoadedDOM listo (estructura cargada)Ejecutar scripts cuando el HTML ya está listo
loadTodo cargado (incluye imágenes, CSS, etc.)Confirmar que los recursos están listos
unloadAl abandonar la página (obsoleto en muchos casos)Intentar guardar datos antes de salir (no recomendado)
errorCuando un recurso falla al cargarDetectar fallos de carga en imágenes o scripts
  • HTML
  • JavaScript
<h2 id="titulo-pagina">Panel de avisos</h2> 
document.addEventListener("DOMContentLoaded", () => { 
  const titulo = document.querySelector("#titulo-pagina"); 
  titulo.classList.add("cargado"); 
});
en este ejemplo

Cuando el DOM está listo, se selecciona el título y se añade una clase. Esto evita errores por intentar acceder antes de tiempo.

5️⃣ Otros eventos útiles​

Además de los anteriores, hay eventos muy comunes en interfaces modernas, especialmente relacionados con scroll, redimensionado de ventana y portapapeles.

EventoDescripciónEjemplo de uso
scrollDesplazamiento de la página/contendorMostrar botón “Subir” o cargar más contenido
resizeCambio de tamaño de la ventanaAjustar elementos o recalcular layouts
  • HTML
  • JavaScript
<p id="estado">Desplázate para activar el evento</p> 
const estado = document.querySelector("#estado"); 
window.addEventListener("scroll", () => {
estado.textContent = "Se ha detectado scroll";
});
en este ejemplo

Al hacer scroll en la ventana, se actualiza el texto del párrafo indicando que el evento se ha activado.

Resumen rápido
  • Ratón: interacción con clics y movimiento del puntero
  • Teclado: pulsaciones de teclas
  • Formularios: escritura, cambios y envío
  • Carga: ejecución cuando el DOM está listo
  • Otros: scroll, resize y acciones del sistema
