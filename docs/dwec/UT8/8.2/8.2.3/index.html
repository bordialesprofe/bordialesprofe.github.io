8.2.3 Objeto Event

Cuando ocurre un evento, JavaScript crea automáticamente un objeto con información sobre lo que ha pasado. Ese objeto se suele llamar event o e y permite conocer detalles como el elemento que lanzó el evento, la tecla pulsada, la posición del ratón o incluso cancelar el comportamiento por defecto.

1️⃣ Recibir el objeto Event en el listener​

Para acceder a la información del evento, se añade un parámetro a la función del addEventListener(). Ese parámetro contendrá el objeto Event.

Sintaxis
elemento.addEventListener("tipoEvento", (event) => { ... })
  • HTML
  • JavaScript
<button id="btn-info">Ver info</button> 
const boton = document.querySelector("#btn-info");

boton.addEventListener("click", (e) => {
console.log(e.type);
});
en este ejemplo

Al hacer clic, se imprime en consola el tipo de evento (click) usando e.type.

2️⃣ target: saber qué elemento provocó el evento​

event.target indica el elemento exacto donde se originó el evento. Es clave cuando hay elementos anidados o cuando se usa delegación (se verá más adelante).

Sintaxis
event.target
  • HTML
  • JavaScript
<article class="tarjeta"> 
  <button class="btn-accion">Acción</button> 
</article> 
const boton = document.querySelector(".btn-accion");

boton.addEventListener("click", (e) => {
console.log(e.target.className);
});
en este ejemplo

Al hacer clic, e.target apunta al botón pulsado y permite consultar su clase.

3️⃣ currentTarget: elemento que escucha el evento​

event.currentTarget es el elemento al que se le ha asignado el listener. A veces coincide con target, pero no siempre (sobre todo con delegación).

Sintaxis
event.currentTarget
  • HTML
  • JavaScript
<article class="tarjeta" id="tarjeta-1"> 
  <button class="btn-accion">Acción</button> 
</article> 
const tarjeta = document.querySelector("#tarjeta-1");

tarjeta.addEventListener("click", (e) => {
  console.log("target:", e.target.tagName);
  console.log("currentTarget:", e.currentTarget.tagName);
});
en este ejemplo

Si se pulsa el botón, target será el botón, pero currentTarget será la tarjeta, porque el listener está en la tarjeta.

4️⃣ Cancelar el comportamiento por defecto con preventDefault()​

Algunos eventos tienen acciones por defecto (por ejemplo, enviar un formulario recarga la página). Con preventDefault() se evita esa acción.

Sintaxis
event.preventDefault()
¿Cuándo interesa usar preventDefault()?

Se utiliza para impedir acciones automáticas del navegador cuando se quiere controlar el flujo con JavaScript. Ejemplos habituales:

  • Enviar un formulario (submit): evita la recarga de la página para validar datos o enviar por fetch/AJAX.
  • Clic en un enlace (click en <a>): evita la navegación para abrir un modal, mostrar un panel o usar un router por hash.
  • Arrastrar y soltar (dragover / drop): permite aceptar un archivo o un elemento soltado en una zona concreta.
  • Menú contextual (contextmenu): evita el menú del navegador para mostrar uno personalizado.
  • HTML
  • JavaScript
<form id="form-aviso"> 
  <input type="text" placeholder="Título" /> 
  <button type="submit">Guardar</button> 
</form> 
const form = document.querySelector("#form-aviso");

form.addEventListener("submit", (e) => {
  e.preventDefault();
  alert("Formulario capturado sin recargar");
});
en este ejemplo

Se evita la recarga de la página al enviar el formulario usando preventDefault().

5️⃣ key: saber qué tecla se ha pulsado​

En eventos de teclado, el objeto Event incluye información sobre la tecla pulsada, como event.key.

Sintaxis
event.key
  • HTML
  • JavaScript
<input id="buscador" type="text" placeholder="Pulsa Enter..." /> 
const buscador = document.querySelector("#buscador");

buscador.addEventListener("keydown", (e) => {
  if (e.key === "Enter") {
    alert("Has pulsado Enter");
  }
});
en este ejemplo

Se detecta si la tecla pulsada es Enter utilizando e.key.

Idea clave

El objeto Event permite saber qué ha pasado (type), quién lo ha provocado (target) y controlar el comportamiento (por ejemplo, preventDefault()).
