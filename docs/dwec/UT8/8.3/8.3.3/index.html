8.3.3 Estadísticas automáticas

En muchas aplicaciones, además de mostrar una lista, se necesita mostrar información resumida: cuántos elementos hay, cuántos cumplen una condición o cuántos quedan pendientes. Estas estadísticas deben actualizarse automáticamente cuando cambian los datos o cuando se aplica un filtro o búsqueda.

El enfoque más limpio es calcular las estadísticas a partir de los datos y actualizar el DOM cada vez que se renderiza la vista.

1️⃣ Mostrar el total de avisos​

Una estadística básica es el número total de elementos. Se calcula con length y se muestra en un elemento de la interfaz.

Idea
total = avisos.length
  • HTML
  • JavaScript
<p>Total: <span id="total">0</span></p> 
<section id="lista-avisos"></section> 
const avisos = [ { titulo: "Cambio de aula", tipo: "urgente" }, 
                 { titulo: "Reunión de tutores", tipo: "normal" }, 
                 { titulo: "Entrega de autorizaciones", tipo: "urgente" } ];

const lista = document.querySelector("#lista-avisos");
const total = document.querySelector("#total");

function renderizar(datos) {
  lista.innerHTML = "";

  datos.forEach((aviso) => {
    const tarjeta = document.createElement("article");
    tarjeta.classList.add("tarjeta");
    tarjeta.textContent = aviso.titulo;
    lista.append(tarjeta);
  });

  total.textContent = datos.length;
}

renderizar(avisos);
en este ejemplo

Se renderiza la lista y, al final, se actualiza el contador con el número de elementos mostrados.

2️⃣ Contar avisos según una condición​

También se pueden mostrar contadores por tipo, estado o cualquier criterio. Para ello se filtra el array y se cuenta el resultado.

Idea
urgentes = avisos.filter(...).length
  • HTML
  • JavaScript
<p>Urgentes: <span id="urgentes">0</span></p> 
<p>Normales: <span id="normales">0</span></p> 
<section id="lista-avisos"></section> 
const avisos = [ { titulo: "Cambio de aula", tipo: "urgente" }, 
                 { titulo: "Reunión de tutores", tipo: "normal" }, 
                 { titulo: "Entrega de autorizaciones", tipo: "urgente" } ];

const lista = document.querySelector("#lista-avisos");
const urgentes = document.querySelector("#urgentes");
const normales = document.querySelector("#normales");

function renderizar(datos) {
  lista.innerHTML = "";

  datos.forEach((aviso) => {
    const tarjeta = document.createElement("article");
    tarjeta.classList.add("tarjeta");
    tarjeta.textContent = aviso.titulo;
    lista.append(tarjeta);
  });

  urgentes.textContent = datos.filter((a) => a.tipo === "urgente").length;
  normales.textContent = datos.filter((a) => a.tipo === "normal").length;
  }

renderizar(avisos);
en este ejemplo

Después de renderizar, se calculan las estadísticas filtrando el array mostrado y actualizando los contadores.

3️⃣ Reutilizar estadísticas con una función​

Para mantener el código limpio, se puede separar el cálculo de estadísticas en una función que se llame desde renderizar().

Idea
renderizar() → actualizarEstadisticas()
  • JavaScript
function actualizarEstadisticas(datos) { 
  urgentes.textContent = datos.filter((a) => a.tipo === "urgente").length; 
  normales.textContent = datos.filter((a) => a.tipo === "normal").length; 
}

function renderizar(datos) {
  lista.innerHTML = "";

  datos.forEach((aviso) => {
    const tarjeta = document.createElement("article");
    tarjeta.classList.add("tarjeta");
    tarjeta.textContent = aviso.titulo;
    lista.append(tarjeta);
  });

  actualizarEstadisticas(datos);
}
en este ejemplo

Se separa el cálculo de estadísticas para que renderizar() se centre en pintar la lista y delegue el resumen en otra función.

Idea clave

Las estadísticas deben calcularse desde los datos y actualizarse cada vez que cambia lo que se muestra (renderizado, filtros o búsqueda).
