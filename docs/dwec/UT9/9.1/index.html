9.1 Comunicación cliente-servidor en aplicaciones web

Las aplicaciones web modernas no se limitan a mostrar información definida en el HTML. Con frecuencia necesitan obtener datos desde un servidor o enviar información para que sea procesada o almacenada.

Para realizar estas tareas, el navegador debe comunicarse con un servidor web. Esta comunicación se basa en el modelo cliente-servidor, utilizado de forma general en el funcionamiento de Internet.

Comprender este modelo es fundamental para entender cómo las aplicaciones web pueden cargar información dinámica y actualizar su contenido sin recargar completamente la página.

1️⃣ El modelo cliente-servidor​

En una aplicación web intervienen dos elementos principales.

- Cliente Es la aplicación que realiza la petición.

En el desarrollo web suele ser el navegador, que ejecuta HTML, CSS y JavaScript.

- Servidor Es el sistema que recibe las peticiones del cliente, procesa la solicitud y devuelve una respuesta.

El flujo de comunicación básico es el siguiente:

en este ejemplo

En una aplicación web moderna, el proceso habitual de comunicación es el siguiente:

  1. El cliente (navegador) envía una petición HTTP al servidor mediante JavaScript.
  2. El servidor recibe la petición y procesa la solicitud.
  3. El servidor devuelve una respuesta, normalmente en formato JSON.
  4. JavaScript utiliza los datos recibidos para actualizar el contenido del documento.

2️⃣ Peticiones y respuestas HTTP​

La comunicación entre cliente y servidor en la web se realiza mediante el protocolo HTTP.

Cada interacción entre el navegador y el servidor se compone de dos elementos:

  • Petición HTTP El cliente solicita un recurso al servidor.
  • Respuesta HTTP El servidor devuelve el recurso solicitado.

Las peticiones pueden utilizar diferentes métodos, entre los que destacan:

  • GET → solicitar información
  • POST → enviar información

El servidor puede responder con distintos tipos de contenido, como:

  • documentos HTML
  • imágenes o archivos
  • datos estructurados en JSON
Recordatorio: uso de JSON

El formato JSON ya se ha utilizado en unidades anteriores para almacenar datos en localStorage. En este tema se utilizará principalmente para intercambiar información entre el cliente y el servidor.

3️⃣ El problema de las recargas completas de página​

En las páginas web tradicionales, cada vez que se necesitaba nueva información era necesario recargar completamente el documento.

Esto ocurre, por ejemplo:

  • al enviar un formulario
  • al cambiar de página
  • al solicitar nuevos datos

Este comportamiento tiene varios inconvenientes:

  • la página desaparece y vuelve a cargarse
  • se pierde parte del estado de la aplicación
  • la interacción resulta más lenta
Ejemplo: actualización dinámica

En una aplicación web moderna, al añadir un producto al carrito solo se actualiza el contador del carrito, sin recargar toda la página.

Esto es posible gracias a la comunicación asíncrona entre el navegador y el servidor.

4️⃣ Comunicación asíncrona​

Para evitar estas recargas completas, las aplicaciones web utilizan comunicación asíncrona entre cliente y servidor.

Esto permite que JavaScript envíe peticiones al servidor mientras la página sigue funcionando con normalidad. Cuando llega la respuesta, el código puede actualizar solo la parte necesaria de la interfaz.

Este enfoque permite crear aplicaciones web más dinámicas y rápidas.

Ejemplo: búsqueda en tiempo real

En muchos buscadores de productos, los resultados se actualizan mientras el usuario escribe.

Cada nueva letra genera una petición al servidor que devuelve resultados actualizados sin necesidad de recargar la página.
