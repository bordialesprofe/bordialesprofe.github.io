9.2 Asincronía en JavaScript

En JavaScript no todas las operaciones se ejecutan necesariamente de forma inmediata. Algunas tareas pueden requerir tiempo para completarse, como esperar a una respuesta del servidor o ejecutar un temporizador.

Para gestionar este tipo de situaciones, JavaScript utiliza mecanismos que permiten continuar la ejecución del programa mientras se espera el resultado de una operación. Este comportamiento se conoce como asincronía.

Comprender este concepto es fundamental para poder realizar peticiones al servidor desde JavaScript, ya que estas operaciones no se resuelven de forma instantánea.

1️⃣ Ejecución síncrona y asíncrona​

En una ejecución síncrona, las instrucciones se ejecutan una detrás de otra, en el mismo orden en que aparecen en el código.

Ejemplo ejecución síncrona
console.log("Inicio");
console.log("Proceso");
console.log("Fin");
en este ejemplo

La salida en consola será:

  • Inicio
  • Proceso
  • Fin

Cada línea se ejecuta únicamente cuando la anterior ha terminado.

En cambio, en una ejecución asíncrona, ciertas operaciones pueden ejecutarse en segundo plano, permitiendo que el programa continúe mientras se espera el resultado.

Un ejemplo sencillo es el uso de temporizadores.

Ejemplo ejecución asíncrona
console.log("Inicio");

setTimeout(() => {
   console.log("Proceso terminado");
}, 2000);

console.log("Fin");
en este ejemplo

La salida en consola será:

  • Inicio
  • Fin
  • Proceso terminado

El temporizador tarda dos segundos en ejecutarse, pero el programa no se detiene esperando ese resultado.

2️⃣ Operaciones que requieren espera​

Existen muchas operaciones en una aplicación web que no pueden completarse de forma inmediata.

Algunos ejemplos habituales son:

  • peticiones a un servidor web
  • lectura o escritura de archivos
  • ejecución de temporizadores
  • acceso a recursos externos

Estas operaciones pueden tardar desde milisegundos hasta varios segundos, dependiendo de factores como la red o la carga del servidor.

Ejemplo del tablón: operación que requiere espera

Cuando una aplicación solicita datos a un servidor mediante una petición HTTP, el navegador debe esperar a que el servidor procese la solicitud y devuelva una respuesta.

Durante ese tiempo, la aplicación no debe bloquearse, sino seguir funcionando con normalidad.

Por esta razón, JavaScript utiliza mecanismos que permiten gestionar este tipo de tareas sin detener la ejecución del programa.

3️⃣ Callbacks​

Uno de los primeros mecanismos utilizados para gestionar operaciones asíncronas en JavaScript fue el uso de callbacks.

Un callback es una función que se pasa como argumento a otra función y que se ejecutará cuando una operación haya terminado.

Ejemplo
function saludar(nombre, callback) {
   console.log("Hola " + nombre);
   callback();
}

saludar("Ana", () => {
   console.log("Operación finalizada");
});
en este ejemplo

Se define una función llamada saludar que recibe dos parámetros: nombre y callback.

  • Primero, la función muestra un mensaje de saludo utilizando el nombre recibido.
  • Después ejecuta la función almacenada en callback.

Al llamar a la función saludar, se pasan dos argumentos:

  • El nombre "Ana".
  • Una función que muestra el mensaje "Operación finalizada".

Cuando se ejecuta el programa ocurre lo siguiente:

  • 1. Se llama a saludar("Ana", ...).
  • 2. La función imprime Hola Ana.
  • 3. A continuación se ejecuta la función callback.
  • 4. La función callback imprime Operación finalizada.

La salida en consola sería:

Hola Ana

Operación finalizada

En este ejemplo La ejecución sigue siendo síncrona. Las instrucciones dentro de saludar se ejecutan en orden: primero se muestra el saludo y después se ejecuta la función callback.

El objetivo del ejemplo es únicamente mostrar que una función puede recibir otra función como argumento y ejecutarla posteriormente. En situaciones reales, los callbacks suelen utilizarse para ejecutar código cuando finaliza una operación asíncrona, como un temporizador o una petición al servidor.

Problema: callbacks anidados

Cuando varias operaciones asíncronas dependen unas de otras, el uso de callbacks puede generar estructuras muy anidadas conocidas como callback hell, que dificultan la comprensión del código.

4️⃣ Promesas​

Para mejorar la gestión de operaciones asíncronas, JavaScript introduce el concepto de promesa (Promise).

Una promesa es un objeto que representa el resultado futuro de una operación asíncrona. Permite indicar qué debe ocurrir cuando dicha operación se complete correctamente o cuando se produzca un error.

Cuando se inicia una operación asíncrona, JavaScript crea una promesa que actúa como un contenedor del resultado de esa operación. Mientras la operación se está ejecutando, la promesa permanece en estado pendiente (pending).

Una vez que la operación termina, la promesa cambia a uno de los siguientes estados:

  • fulfilled → la operación se ha completado correctamente y devuelve un resultado.
  • rejected → la operación ha producido un error.

Para trabajar con el resultado de una promesa se utilizan principalmente dos métodos:

  • .then(), que se ejecuta cuando la promesa se resuelve correctamente.
  • .catch(), que se utiliza para gestionar posibles errores.

Este mecanismo permite organizar el código de forma más clara que utilizando múltiples callbacks anidados y facilita la gestión de operaciones cuyo resultado se conocerá más adelante, como ocurre al realizar peticiones al servidor.

En el siguiente ejemplo se muestra cómo crear una promesa y cómo gestionar su resultado.

Ejemplo uso de promesas
const promesa = new Promise((resolve, reject) => {
   let operacionCorrecta = true;

   if (operacionCorrecta) {
      resolve("Operación completada");
   } else {
      reject("Error en la operación");
   }
});

promesa
   .then(resultado => console.log(resultado))
   .catch(error => console.log(error));
en este ejemplo

Se crea una promesa mediante new Promise(). El constructor de la promesa recibe una función con dos parámetros: resolve y reject.

  • resolve() se utiliza cuando la operación se completa correctamente.
  • reject() se utiliza cuando se produce un error.

Dentro de la promesa se simula una operación mediante la variable operacionCorrecta. Si la operación es correcta, se ejecuta resolve("Operación completada"). En caso contrario se ejecuta reject("Error en la operación").

Después se utiliza el método .then() para indicar qué hacer cuando la promesa se resuelve correctamente. En este caso se muestra el resultado en la consola.

También se utiliza .catch() para capturar posibles errores si la promesa es rechazada.

Cuando se ejecuta el programa ocurre lo siguiente:

  1. Se crea la promesa.
  2. Se evalúa la variable operacionCorrecta.
  3. Si es true, se ejecuta resolve.
  4. Si es false, se ejecuta reject.
  5. El método .then() procesa el resultado correcto.
  6. El método .catch() gestionaría el error si se produjera.

Las promesas permiten organizar mejor el código y evitar la anidación excesiva de callbacks.

5️⃣ Uso de async y await​

La forma más moderna y clara de trabajar con operaciones asíncronas en JavaScript es mediante funciones async y el operador await.

LUna función declarada con async devuelve automáticamente una promesa. Esto significa que su resultado puede utilizarse como el resultado de una operación asíncrona.

Dentro de una función async es posible utilizar el operador await. Este operador permite esperar el resultado de una promesa antes de continuar con la ejecución del código.

Gracias a async y await, el código puede escribirse de una forma muy similar al código síncrono, lo que facilita su lectura y mantenimiento.

Este enfoque es muy habitual al trabajar con peticiones HTTP mediante fetch(), ya que permite procesar la respuesta del servidor de forma más clara.

En el siguiente ejemplo se muestra cómo utilizar funciones async y await.

Ejemplo de uso de funciones async y await
async function obtenerMensaje() {
   return "Operación completada";
}

async function mostrarMensaje() {
   const mensaje = await obtenerMensaje();
   console.log(mensaje);
}

mostrarMensaje();
en este ejemplo

Se definen dos funciones asíncronas: obtenerMensaje y mostrarMensaje.

La función obtenerMensaje está declarada con async, por lo que su resultado se devuelve en forma de promesa. En este caso la función devuelve el texto "Operación completada".

La función mostrarMensaje también es una función async. Dentro de ella se utiliza el operador await para esperar el resultado de la función obtenerMensaje.

Cuando se ejecuta await obtenerMensaje(), el programa espera a que la promesa se resuelva y devuelve el valor "Operación completada", que se almacena en la variable mensaje.

Finalmente, el mensaje se muestra en la consola mediante console.log.

Cuando se ejecuta el programa ocurre lo siguiente:

  1. Se llama a la función mostrarMensaje().
  2. Dentro de la función se ejecuta await obtenerMensaje().
  3. La función obtenerMensaje devuelve el texto "Operación completada".
  4. El valor se guarda en la variable mensaje.
  5. El mensaje se muestra en la consola.

La salida en consola sería: Operación completada

Con este enfoque, el código se escribe de una forma muy similar al código síncrono, pero mantiene el comportamiento asíncrono.

Idea clave: async y await

Las peticiones al servidor que se realizarán en los siguientes apartados utilizan promesas.

Por este motivo es habitual trabajar con async y await para gestionar estas operaciones de forma más clara.
