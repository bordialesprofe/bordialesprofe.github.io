9.3 Peticiones HTTP desde JavaScript

Las aplicaciones web modernas necesitan comunicarse con servidores para obtener o enviar información. Hasta hace unos años, este tipo de comunicación se realizaba principalmente mediante recargas completas de página.

Sin embargo, JavaScript permite realizar peticiones HTTP directamente desde el navegador, lo que permite intercambiar datos con el servidor sin necesidad de recargar el documento.

Este mecanismo es la base de muchas aplicaciones web actuales, como buscadores dinámicos, sistemas de comentarios, paneles de administración o aplicaciones que consumen APIs externas.

1️⃣ Concepto de AJAX​

El término AJAX proviene de Asynchronous JavaScript and XML y hace referencia a una técnica que permite realizar peticiones asíncronas al servidor desde JavaScript.

Aunque el nombre incluye XML, en la práctica actualmente se utilizan principalmente datos en formato JSON.

El objetivo de AJAX es permitir que una página web pueda:

  • enviar peticiones al servidor
  • recibir datos como respuesta
  • actualizar parte del contenido del documento

todo ello sin recargar la página completa.

Concepto clave: AJAX

AJAX no es una tecnología concreta, sino una técnica de desarrollo que combina JavaScript, HTTP y el intercambio de datos entre cliente y servidor.

2️⃣ Solicitudes desde el navegador​

Cuando una aplicación web realiza una petición mediante JavaScript, el navegador actúa como cliente, enviando una solicitud al servidor, que la procesa y devuelve una respuesta.

El proceso habitual es el siguiente:

  1. JavaScript envía una petición HTTP al servidor.
  2. El servidor procesa la solicitud.
  3. El servidor devuelve una respuesta, normalmente en formato JSON.
  4. JavaScript procesa los datos recibidos y actualiza el contenido de la página.

Este tipo de comunicación permite construir aplicaciones mucho más dinámicas.

Ejemplo del tablón: carga y actualización de avisos

En la aplicación del tablón, el navegador puede solicitar al servidor la lista de avisos publicados.

  1. El navegador envía una petición al servidor solicitando los avisos.
  2. El servidor consulta su base de datos o su fuente de datos.
  3. El servidor devuelve los avisos en formato JSON.
  4. JavaScript genera dinámicamente los elementos del tablón y los inserta en la página.

Este proceso puede repetirse periódicamente para comprobar si existen nuevos avisos, permitiendo actualizar el contenido del tablón sin necesidad de recargar la página.

3️⃣ La API Fetch​

Actualmente, la forma más utilizada de realizar peticiones HTTP desde JavaScript es mediante la API Fetch.

La función fetch() permite enviar peticiones al servidor y devuelve una promesa con la respuesta recibida.

Ejemplo básico de petición
fetch("avisos.php")
  .then(respuesta => respuesta.json())
  .then(datos => {
      console.log(datos);
  });
en este ejemplo
  • fetch("avisos.php") envía una petición HTTP al servidor solicitando los avisos del tablón.
  • El servidor ejecuta el archivo avisos.php, que obtiene los avisos almacenados en el sistema.
  • El servidor devuelve la información en formato JSON.
  • respuesta.json() convierte la respuesta recibida en un objeto JavaScript.
  • La variable datos contiene la lista de avisos, que en este ejemplo se muestra en la consola.
Ejemplo del tablón: carga de avisos

En la aplicación del tablón, una página puede solicitar al servidor la lista de avisos publicados mediante una petición HTTP.

Cuando el servidor devuelve los datos, JavaScript puede utilizarlos para generar dinámicamente los elementos del tablón y mostrarlos en la página sin necesidad de recargar el documento.

Recomendación: async/await con fetch

Aunque las peticiones realizadas con fetch() pueden gestionarse mediante .then(), en aplicaciones modernas es habitual utilizar async y await, ya que permiten escribir el código de forma más clara y fácil de leer.
