9.4 Uso de Fetch para obtener datos del servidor

La API Fetch permite realizar peticiones HTTP desde JavaScript y trabajar con la respuesta recibida de forma sencilla.

Cuando se utiliza fetch(), el navegador envía una petición al servidor y devuelve una promesa que representa la respuesta. Una vez recibida la respuesta, JavaScript puede procesarla y utilizar los datos para actualizar la página.

Este mecanismo es especialmente útil para cargar información desde el servidor sin recargar el documento completo.

Recordatorio: métodos HTTP GET y POST

Las peticiones HTTP pueden utilizar distintos métodos según el tipo de operación que se quiera realizar.

MétodoUso principalEjemplo
GETSolicitar información al servidorCargar los avisos del tablón
POSTEnviar datos al servidorPublicar un nuevo aviso

Las peticiones GET se utilizan cuando una aplicación necesita obtener datos, mientras que las peticiones POST se utilizan cuando se deben enviar datos para ser procesados o almacenados en el servidor.

En esta unidad se utilizarán ambos métodos mediante la función fetch().

1️⃣ Realizar una petición GET​

Una de las operaciones más habituales en una aplicación web consiste en solicitar información al servidor. Para ello se utiliza normalmente una petición HTTP GET.

Cuando JavaScript realiza una petición GET mediante fetch(), el navegador envía una solicitud al servidor indicando el recurso que se desea obtener. El servidor procesa la petición y devuelve una respuesta con la información solicitada.

La función fetch() devuelve una promesa que representa la respuesta del servidor.

En el siguiente ejemplo, la aplicación solicita al servidor la lista de avisos publicados en el tablón.

Ejemplo
fetch("avisos.php")
  .then(respuesta => respuesta.json())
  .then(avisos => {
     console.log(avisos);
  });
en este ejemplo

La aplicación solicita al servidor la lista de avisos del tablón. El servidor responde con los datos en formato JSON y JavaScript los recibe en la variable avisos, que posteriormente podrían utilizarse para mostrar los avisos en la página.

Cuando se ejecuta este código ocurre lo siguiente:

  1. fetch("avisos.php") envía una petición HTTP al servidor.
  2. El servidor ejecuta el archivo avisos.php.
  3. El servidor devuelve los datos de los avisos en formato JSON.
  4. respuesta.json() convierte la respuesta en un objeto JavaScript.
  5. La variable avisos contiene la información recibida.

2️⃣ Procesar la respuesta del servidor​

Cuando se realiza una petición mediante fetch(), el servidor devuelve una respuesta HTTP. Esta respuesta no contiene directamente los datos que se desean utilizar, sino que se encapsula dentro de un objeto denominado Response.

El objeto Response contiene información sobre la respuesta enviada por el servidor, como por ejemplo:

  • el contenido devuelto por el servidor
  • el estado de la respuesta
  • las cabeceras HTTP Para trabajar con esta información, fetch() devuelve una promesa que recibe como parámetro un objeto Response.
Ejemplo
fetch("avisos.php")
  .then(respuesta => {
     console.log(respuesta);
     console.log(respuesta.status);
     console.log(respuesta.ok);
  });
en este ejemplo

Se realiza una petición al servidor mediante fetch().

La función fetch() devuelve una promesa que recibe un objeto Response, representado en el código mediante la variable respuesta.

Dentro de la función se muestran algunas propiedades del objeto:

  • respuesta → objeto completo devuelto por el servidor
  • respuesta.status → código de estado HTTP de la respuesta
  • respuesta.ok → indica si la petición se ha realizado correctamente

Para obtener los datos que devuelve el servidor es necesario utilizar alguno de los métodos del objeto Response, como:

  • response.json() → para respuestas en formato JSON
  • response.text() → para respuestas en texto plano

Estos métodos permiten extraer el contenido de la respuesta y convertirlo en un formato que pueda utilizarse en JavaScript.

3️⃣ Conversión de respuestas a JSON​

En muchas aplicaciones web modernas, el servidor devuelve los datos en formato JSON. Para poder utilizar esa información en JavaScript es necesario convertir la respuesta del servidor en un objeto JavaScript.

El objeto Response proporciona el método json(), que permite leer el contenido de la respuesta y convertirlo automáticamente a un objeto JavaScript.

Ejemplo
fetch("avisos.php")
  .then(respuesta => respuesta.json())
  .then(avisos => {
     console.log(avisos);
  });
en este ejemplo

En la aplicación del tablón, la página puede solicitar al servidor la lista de avisos publicados.

Cuando el servidor devuelve los datos en formato JSON, JavaScript puede utilizarlos para generar dinámicamente los elementos del tablón y mostrarlos en la página sin necesidad de recargar el documento.

En este ejemplo se realiza una petición al servidor mediante fetch().

  1. fetch("avisos.php") envía una petición HTTP al servidor solicitando los avisos del tablón.
  2. El servidor responde con los datos en formato JSON.
  3. respuesta.json() lee el contenido de la respuesta y lo convierte en un objeto JavaScript.
  4. La variable avisos contiene los datos recibidos del servidor.

Una vez convertidos los datos, pueden utilizarse en la aplicación para mostrar información o actualizar el contenido de la página.

El archivo avisos.php podría generar una respuesta en formato JSON como la siguiente:

Ejemplo de respuesta en PHP
<?php
$avisos = [
  ["titulo" => "Excursión fin de curso", "autor" => "Dirección"],
  ["titulo" => "Reunión de familias", "autor" => "Jefatura de estudios"]
];

header("Content-Type: application/json");
echo json_encode($avisos);
?>
en este ejemplo

Este script devuelve los avisos en formato JSON, que posteriormente JavaScript podrá convertir en un objeto mediante response.json().

4️⃣ Uso de async y await con fetch​

Aunque las peticiones realizadas con fetch() pueden gestionarse mediante .then(), en aplicaciones modernas es habitual utilizar async y await, ya que permiten escribir el código de forma más clara y fácil de leer.

Cuando se utiliza await, la ejecución del programa espera a que la promesa se resuelva antes de continuar, lo que permite escribir el código de una forma muy similar a la programación síncrona.

Ejemplo
async function cargarAvisos() {
   const respuesta = await fetch("avisos.php");
   const avisos = await respuesta.json();

   console.log(avisos);
}
cargarAvisos();
en este ejemplo

En la aplicación del tablón, la función cargarAvisos() podría ejecutarse cuando se carga la página.

De esta forma, la aplicación solicitaría los avisos al servidor y actualizaría el contenido del tablón automáticamente sin necesidad de recargar la página.

En este ejemplo se define una función asíncrona llamada cargarAvisos.

  1. fetch("avisos.php") envía una petición HTTP al servidor solicitando los avisos del tablón.
  2. await espera a que el servidor devuelva la respuesta.
  3. La variable respuesta contiene un objeto Response.
  4. respuesta.json() convierte el contenido de la respuesta en un objeto JavaScript.
  5. La variable avisos contiene los datos recibidos del servidor.

Finalmente, los avisos se muestran en la consola.

El uso de async y await permite escribir código asíncrono de forma más clara y legible que utilizando múltiples llamadas a .then().

5️⃣ Manejo básico de errores​

Cuando una aplicación realiza una petición al servidor pueden producirse distintos tipos de errores. Por ejemplo:

  • el servidor puede no estar disponible
  • la conexión de red puede fallar
  • el servidor puede devolver una respuesta incorrecta

Por este motivo es importante que las aplicaciones web incluyan mecanismos básicos de control de errores al realizar peticiones mediante fetch().

🔶 Cuando se trabaja con promesas utilizando .then(), los errores pueden gestionarse mediante el método .catch().

Ejemplo manejo errores con .then()
fetch("avisos.php")
  .then(respuesta => respuesta.json())
  .then(avisos => {
     console.log(avisos);
  })
  .catch(error => {
     console.error("Error en la petición:", error);
  });
en este ejemplo

Se realiza una petición al servidor mediante fetch().

Si la petición se realiza correctamente, la respuesta se convierte a JSON y los datos se muestran en la consola.

Si se produce algún error durante la petición, el método .catch() captura el error y muestra un mensaje en la consola.

🔶 Cuando se utiliza async y await, los errores pueden gestionarse mediante una estructura try...catch.

Ejemplo manejo errores asyn y await
async function cargarAvisos() {
   try {
      const respuesta = await fetch("avisos.php");
      const avisos = await respuesta.json();
      console.log(avisos);
   } catch(error) {
      console.error("Error al cargar los avisos:", error);
   }
}
cargarAvisos();
en este ejemplo

En este ejemplo la función cargarAvisos realiza una petición al servidor utilizando fetch().

Si la petición se ejecuta correctamente, los datos recibidos se convierten a JSON y se muestran en la consola.

Si se produce algún error durante la petición, la estructura try...catch permite capturar el error y ejecutar el código indicado dentro de catch.

Este control básico de errores permite que la aplicación continúe funcionando incluso si se produce algún problema durante la comunicación con el servidor.

Buenas prácticas: gestión de errores

En aplicaciones web reales es recomendable gestionar los errores de forma adecuada, mostrando mensajes informativos al usuario o registrando los errores para su análisis.
