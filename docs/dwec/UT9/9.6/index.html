9.6 Envío de datos al servidor

Hasta ahora se ha visto cómo una aplicación web puede obtener información desde el servidor mediante peticiones HTTP. Sin embargo, en muchas aplicaciones también es necesario enviar datos al servidor para que estos sean procesados o almacenados.

Esto ocurre en situaciones como:

  • publicar un aviso en el tablón
  • enviar un formulario de registro
  • añadir un comentario
  • actualizar información en una base de datos

En las páginas web tradicionales, estos datos se envían mediante formularios HTML, utilizando los métodos HTTP GET o POST.

Cuando se trabaja con JavaScript, el envío de datos también puede realizarse mediante la función fetch(), utilizando normalmente peticiones HTTP POST.

Formulario o JavaScript → petición POST → servidor → procesamiento de datos

1️⃣ Peticiones POST con fetch​

Para enviar datos al servidor mediante fetch(), es necesario indicar el método HTTP que se desea utilizar.

Esto se realiza mediante un objeto de configuración que se pasa como segundo parámetro a la función fetch().

fetch("nuevoAviso.php", {
   method: "POST"
});
en este ejemplo

Se envía una petición al servidor utilizando el método HTTP POST.

El navegador envía la solicitud al archivo nuevoAviso.php, que se ejecutará en el servidor para procesar la información recibida.

Recordatorio: método POST

El método HTTP POST se utiliza para enviar información al servidor con el objetivo de que sea procesada o almacenada.

Es el método que utilizan habitualmente los formularios cuando se envían datos al servidor.

2️⃣ Envío de datos con FormData​

Cuando se desean enviar varios campos de información, se puede utilizar el objeto FormData.

Este objeto permite construir un conjunto de datos similar al que enviaría un formulario HTML.

const datos = new FormData();

datos.append("titulo", "Excursión fin de curso");
datos.append("autor", "Dirección");
en este ejemplo

Se crea un objeto FormData llamado datos.

El método append() permite añadir campos al objeto indicando:

  • el nombre del campo
  • el valor que se desea enviar

Estos datos se enviarán posteriormente al servidor como parte de la petición HTTP.

Ventaja de FormData

El objeto FormData permite enviar datos al servidor utilizando el mismo formato que emplean los formularios HTML.

Esto facilita que el servidor pueda leer los datos sin necesidad de realizar cambios adicionales.

Envío de archivos con FormData

El objeto FormData también permite enviar archivos al servidor, por ejemplo cuando un formulario incluye un campo de tipo file.

En ese caso, el archivo se añade al objeto FormData igual que cualquier otro campo y se envía mediante fetch().

Este mecanismo es el que utilizan muchas aplicaciones web para subir imágenes, documentos u otros archivos al servidor.

3️⃣ Envío completo de datos al servidor​

Una vez creado el objeto FormData, puede enviarse al servidor mediante fetch() utilizando una petición POST.

const datos = new FormData();

datos.append("titulo", "Excursión fin de curso");
datos.append("autor", "Dirección");

fetch("nuevoAviso.php", {
   method: "POST",
   body: datos
});
en este ejemplo

En este ejemplo se envían los datos al servidor mediante una petición POST.

La propiedad body contiene el objeto FormData, que incluye los campos que se desean enviar.

Cuando el servidor recibe la petición, podrá acceder a esos campos para procesarlos.

4️⃣ Lectura de los datos en el servidor (PHP)​

Cuando los datos se envían mediante una petición POST, el servidor puede acceder a ellos utilizando la variable $_POST en PHP.

<?php
$titulo = $_POST["titulo"];
$autor = $_POST["autor"];

echo "Aviso recibido: " . $titulo;
?>
en este ejemplo

En este ejemplo el archivo nuevoAviso.php recibe los datos enviados desde JavaScript.

La variable $_POST permite acceder a los valores enviados en la petición HTTP.

Cada campo enviado mediante FormData puede recuperarse utilizando su nombre correspondiente.

Ejemplo del tablón: publicación de avisos

En la aplicación del tablón, el usuario puede rellenar un formulario para publicar un nuevo aviso.

JavaScript puede recoger los datos introducidos por el usuario y enviarlos al servidor mediante una petición POST.

El servidor procesa la información y almacena el nuevo aviso, que posteriormente podrá mostrarse en el tablón junto con los demás anuncios.
