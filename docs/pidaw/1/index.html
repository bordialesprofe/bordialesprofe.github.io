1. Proyecto Marco del módulo

1.1 ¿En qué consiste el Proyecto Marco?​

El Proyecto Marco es la estructura común que guía el desarrollo de tu proyecto individual durante el módulo PIDAW. Define el alcance técnico mínimo, las características que debe cumplir cualquier aplicación desarrollada y las fases principales del proceso de trabajo.

Aunque cada estudiante creará una aplicación diferente, todos los proyectos deben ajustarse a esta base común para garantizar que se alcanzan los resultados de aprendizaje del módulo y que el trabajo realizado es comparable, evaluable y realista.

El Proyecto Marco actúa como una referencia clara que te permitirá avanzar de manera ordenada, sabiendo qué se espera en cada fase y qué requisitos debe cumplir tu solución final.

1.2. Objetivos generales del Proyecto Marco​

Mediante este proyecto se busca que puedas:

  • Aplicar los conocimientos adquiridos en los módulos del ciclo para resolver un problema real.
  • Planificar, diseñar y desarrollar una aplicación web con una estructura profesional.
  • Integrar aspectos técnicos, organizativos y de calidad en un único producto.
  • Generar documentación técnica y una memoria final coherente con el proceso seguido.
  • Mejorar tu autonomía, tu capacidad de análisis y tu criterio técnico.

1.3 Alcance obligatorio de la aplicación​

Toda aplicación desarrollada en el PIDAW debe incluir, como mínimo:

🖥️ Interfaz accesible​

Tu aplicación debe presentar una interfaz clara, coherente y fácil de usar.
Esto implica que cuides aspectos como:

  • la navegación,
  • la consistencia visual,
  • el diseño adaptable a distintos dispositivos,
  • y los criterios básicos de accesibilidad (contraste, etiquetas, orden de lectura).

El usuario debe poder moverse por tu aplicación sin dificultades ni confusiones.

🛢️ Base de datos remota​

Deberás almacenar y recuperar la información desde una base de datos alojada en un servidor.
Esto significa que tendrás que:

  • diseñar las tablas necesarias,
  • establecer la conexión desde tu aplicación,
  • realizar consultas seguras,
  • controlar errores,
  • y garantizar que los datos se guardan correctamente.

No basta con usar almacenamiento local del navegador.

✔️ Validación​

Todos los formularios de tu aplicación deben validar los datos tanto en cliente como en servidor.
Se espera que:

  • evites envíos con datos vacíos o incorrectos,
  • controles formatos como correos, fechas o números,
  • muestres mensajes de error claros,
  • y vuelvas a validar en el servidor por seguridad.

Así garantizas que los datos que llegan a tu aplicación sean fiables.

📁 CRUD completo​

Deberás implementar un CRUD sobre una entidad principal de tu proyecto (por ejemplo: tareas, reservas, productos…).
Esto significa que tu aplicación debe permitir:

  • crear nuevos registros,
  • leer y mostrar datos en listados,
  • actualizar información mediante formularios,
  • eliminar registros de manera segura.

Si hay muchos datos, deberás incluir filtros o paginación básica.

🔐 Autenticación​

Tu proyecto debe incluir un sistema de registro e inicio de sesión.
Esto implica que tengas que:

  • crear cuentas nuevas,
  • proteger las contraseñas,
  • validar credenciales en el servidor,
  • restringir el acceso a zonas privadas,
  • y permitir el cierre de sesión.

El objetivo es que controles qué puede ver cada usuario.

🧪 Pruebas y documentación​

Tendrás que comprobar que tu aplicación funciona correctamente mediante pruebas básicas.
Esto incluye:

  • revisar que cada funcionalidad responde como debe,
  • detectar y corregir errores,
  • realizar pruebas de accesibilidad,
  • y documentar las incidencias.

Además, deberás elaborar:

  • un manual de usuario,
  • y una documentación técnica sobre cómo has construido tu proyecto.

☁️ Despliegue​

Tu proyecto debe estar publicado en un servidor accesible desde Internet.
Para ello tendrás que:

  • preparar tu aplicación para producción,
  • configurar la conexión a la base de datos online,
  • subir el proyecto al servidor o plataforma correspondiente,
  • comprobar que funciona correctamente en remoto,
  • y obtener una URL pública que puedas mostrar en tu defensa.

El despliegue es el paso final antes de presentar tu trabajo.

Este alcance no limita tu creatividad: puedes añadir más funcionalidades si dispones del tiempo y del nivel técnico necesario, siempre después de completar los requisitos mínimos.

1.4 Requisitos técnicos mínimos​

Tu aplicación deberá desarrollarse empleando:

  • Tecnologías del lado cliente estudiadas en DWEC.
  • Tecnologías del lado servidor trabajadas en DWES.
  • Diseño e implementación de la interfaz conforme a lo visto en DIW.
  • Almacenamiento de datos mediante los contenidos de Bases de Datos.

A partir de esta base, puedes añadir librerías, estilos adicionales o pequeñas mejoras siempre que no supongan una complejidad que impida finalizar el proyecto dentro de las horas del módulo.

1.5 Qué se espera de ti durante el desarrollo​

A lo largo del módulo deberás:

  • Avanzar siguiendo las fases del Proyecto Marco.
  • Trabajar de forma autónoma tu proyecto individual.
  • Participar en las actividades colaborativas de revisión y mejora.
  • Entregar las evidencias correspondientes a cada fase.
  • Aplicar las recomendaciones recibidas durante el proceso.
  • Mantener ordenado tu repositorio y tu documentación técnica.

El objetivo no es solo crear una aplicación, sino aprender a organizar un proyecto, tomar decisiones fundamentadas y avanzar de manera estructurada, como se haría en un entorno profesional.
