Fase 3 - Desarrollo

El objetivo de esta fase es desarrollar una primera versión funcional del proyecto (MVP – Minimum Viable Product) implementando las funcionalidades principales definidas en la fase de análisis y diseño.

Se busca que el sistema pueda utilizarse de forma básica, aunque todavía no incluya todas las características previstas ni un diseño completamente final.

Durante esta fase se trabajará principalmente en:

  • Implementación del frontend
  • Desarrollo de la lógica de la aplicación
  • Integración con servicios o base de datos
  • Desarrollo de las funcionalidades principales del sistema

No se pretende que el sistema esté terminado, sino que permita demostrar que la idea funciona y que el sistema puede utilizarse.

1️⃣ Qué debes hacer en esta fase​

1) Preparar el entorno de desarrollo del proyecto

Se debe configurar el entorno de trabajo necesario para el desarrollo del sistema. Esto incluye:

  • Crear el repositorio del proyecto.
  • Organizar la estructura de carpetas y archivos.
  • Configurar las herramientas de desarrollo que se vayan a utilizar.
  • Establecer una forma de trabajo organizada durante el desarrollo.

2) Implementar la interfaz básica de la aplicación

Se deben desarrollar las pantallas principales definidas en los wireframes de la fase anterior.

  • Crear la estructura de las páginas o vistas del sistema.
  • Aplicar estilos básicos a la interfaz.
  • Permitir la navegación entre las diferentes pantallas.

La interfaz no necesita ser definitiva, pero debe permitir usar la aplicación de forma clara y comprensible.

3) Desarrollar la lógica principal de la aplicación

Se debe implementar la lógica necesaria para que la aplicación funcione correctamente.

Esto puede incluir, por ejemplo:

  • Gestión de eventos de la interfaz.
  • Validación de formularios.
  • Procesamiento de datos introducidos por el usuario.
  • Interacción entre diferentes partes del sistema.

4) Integrar el sistema con datos o servicios

El proyecto debe poder gestionar información real o simulada para demostrar su funcionamiento.

Dependiendo del proyecto, esto puede implicar:

  • Conexión con una base de datos.
  • Uso de almacenamiento local.
  • Consumo de APIs externas.
  • Gestión de usuarios o contenidos.

5) Implementar las funcionalidades principales del proyecto

Se deben desarrollar las funcionalidades más importantes que definen el valor del proyecto.

Estas funcionalidades deben permitir demostrar que la aplicación resuelve el problema planteado en la fase de análisis.

No es necesario implementar todas las características previstas, pero sí aquellas que permitan mostrar el funcionamiento esencial del sistema.

2️⃣ Entregables de la fase​

Al finalizar esta fase se deberán entregar los siguientes elementos:

📄 Entregable 1 - Repositorio del proyecto​

Se deberá entregar el enlace al repositorio del proyecto, donde se encuentre todo el código desarrollado durante esta fase.

El repositorio debe incluir:

  • Una estructura de carpetas clara y organizada.
  • El código fuente del proyecto.
  • Commits que reflejen el progreso del desarrollo.
  • Un archivo README.md con información básica del proyecto.

El archivo README debe incluir, al menos:

  • Nombre del proyecto.
  • Breve descripción de la aplicación.
  • Tecnologías utilizadas.
  • Instrucciones básicas para ejecutar el proyecto.

El repositorio debe reflejar el desarrollo realizado durante la fase mediante commits que muestren el progreso del proyecto.

📄 Entregable 2 - MVP funcional​

Se deberá presentar una primera versión funcional de la aplicación (MVP).

Esta versión debe permitir:

  • Navegar entre las principales pantallas del sistema.
  • Utilizar las funcionalidades básicas de la aplicación.
  • Interactuar con datos o servicios cuando sea necesario.

El MVP no necesita incluir todas las funcionalidades previstas ni un diseño final completo, pero debe permitir demostrar el funcionamiento esencial del proyecto.

📄 Entregable 3 - Documento breve de desarrollo​

Se deberá entregar un documento breve en formato PDF en el que se describa el desarrollo realizado durante esta fase.

Este documento debe incluir:

  • Descripción general del estado actual del proyecto.
  • Funcionalidades implementadas en el MVP.
  • Tecnologías utilizadas.
  • Breve explicación de la estructura del proyecto.
  • Principales dificultades encontradas durante el desarrollo.

Se recomienda que el documento tenga entre 2 y 4 páginas.

Presentación del progreso del proyecto

Durante esta fase se podrán realizar presentaciones breves al grupo para mostrar el progreso del proyecto.

En estas presentaciones se deberá:

  • Mostrar el funcionamiento del MVP.
  • Explicar las funcionalidades implementadas.
  • Comentar las decisiones tomadas durante el desarrollo.

Estas presentaciones permitirán recibir comentarios y sugerencias que ayuden a mejorar el proyecto antes de las siguientes fases.

3️⃣ Qué partes se incorporan a la memoria final​

El trabajo realizado en esta fase se incorporará principalmente al apartado 5. Desarrollo de la memoria final.

La memoria final recoge todo el proceso del proyecto, por lo que cada fase del módulo aporta contenido a diferentes secciones del documento. En el caso de esta fase, el desarrollo del MVP servirá para documentar cómo se ha construido técnicamente la aplicación.

En este apartado se describirán aspectos como las tecnologías utilizadas, la estructura del código del proyecto y las funcionalidades implementadas durante el desarrollo.

Es recomendable incluir capturas de pantalla del funcionamiento de la aplicación para ilustrar las funcionalidades desarrolladas.

Para conocer con más detalle qué debe incluir este capítulo, consulta el apartado Estructura de la memoria final.

Anexos de la memoria final

Parte del material generado durante el desarrollo también puede incorporarse al apartado 10. Anexos de la memoria final, como por ejemplo:

  • Capturas del funcionamiento del MVP.
  • Diagramas o esquemas del proyecto.
  • Fragmentos de código relevantes.
  • Versiones preliminares de la interfaz.

4️⃣ Sugerencias para completar la fase​

Durante el desarrollo del MVP pueden tenerse en cuenta algunas recomendaciones que facilitan el trabajo:

  • Empieza por una versión muy simple. Primero crea la estructura básica de la aplicación y asegúrate de que se puede navegar entre las pantallas.
  • Implementa las funcionalidades poco a poco. Es mejor tener pocas funciones que funcionen correctamente que muchas incompletas.
  • Prueba la aplicación con frecuencia. Comprobar el funcionamiento mientras desarrollas ayuda a detectar errores antes de que se acumulen.
  • Realiza commits frecuentes en el repositorio. Esto permite guardar el progreso del proyecto y mantener un historial claro de cambios.
  • Guarda capturas de pantalla del desarrollo. Pueden ser útiles posteriormente para documentar el proyecto en la memoria final.
