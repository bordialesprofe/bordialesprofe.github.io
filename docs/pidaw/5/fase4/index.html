Fase 4 - Pruebas y calidad

En esta fase se comprobará que la aplicación desarrollada funciona correctamente y cumple con las funcionalidades previstas. El objetivo es identificar posibles errores, validar el comportamiento del sistema y mejorar la calidad del proyecto antes de su despliegue.

Para ello se realizarán diferentes pruebas sobre la aplicación y se aplicarán las correcciones necesarias para mejorar su funcionamiento, usabilidad y accesibilidad.

1️⃣ Qué debes hacer en esta fase​

En esta fase debes probar el funcionamiento de tu aplicación y corregir los posibles errores detectados.

El objetivo es comprobar que las funcionalidades implementadas en el MVP funcionan correctamente y que la aplicación responde de forma adecuada ante diferentes situaciones de uso.

Durante esta fase se deberán realizar diferentes tipos de pruebas:

1) Pruebas funcionales

Consisten en comprobar que las funcionalidades principales de la aplicación funcionan correctamente.

Por ejemplo:

  • registro o autenticación de usuarios
  • creación, modificación o eliminación de datos
  • navegación entre pantallas
  • uso de formularios o buscadores Debes verificar que cada funcionalidad cumple el comportamiento esperado.

2) Pruebas de validación

Permiten comprobar cómo responde la aplicación ante diferentes situaciones de uso.

Por ejemplo:

  • introducción de datos incorrectos
  • campos obligatorios vacíos
  • datos con formato incorrecto

La aplicación debe responder mostrando mensajes claros o evitando acciones incorrectas.

3) Pruebas de accesibilidad

También debes revisar algunos aspectos básicos de accesibilidad y usabilidad de la aplicación.

Por ejemplo:

  • contraste adecuado entre colores
  • textos alternativos en imágenes
  • navegación clara
  • buena legibilidad del contenido

Estas comprobaciones ayudan a mejorar la experiencia de uso de la aplicación.

Una vez realizadas las pruebas, deberás corregir los errores detectados y mejorar el funcionamiento general del sistema.

2️⃣ Entregables de la fase​

Al finalizar esta fase se deberán entregar los siguientes elementos:

📄 Entregable 1 – Documento de pruebas​

Documento en formato PDF en el que se describan las pruebas realizadas sobre la aplicación.

Debe incluir:

  • listado de pruebas realizadas
  • resultado de cada prueba
  • errores detectados durante las pruebas
  • correcciones realizadas en la aplicación

📄 Entregable 2 - Versión corregida del MVP​

Versión actualizada de la aplicación en la que se hayan corregido los errores detectados durante las pruebas.

La aplicación debe mostrar una mejora respecto a la versión presentada en la fase anterior.

3️⃣ Qué partes se incorporan a la memoria final​

El trabajo realizado en esta fase se incorporará principalmente al Apartado 7. Pruebas de la memoria final.

En este apartado se describirán las pruebas realizadas sobre la aplicación para comprobar su correcto funcionamiento. El trabajo desarrollado durante esta fase servirá para documentar cómo se ha verificado la calidad del sistema antes de su despliegue.

En este capítulo se explicarán los tipos de pruebas realizadas, los errores detectados y las correcciones aplicadas durante el proceso.

Para conocer con más detalle qué debe incluir este capítulo, consulta el apartado Estructura de la memoria final.

Anexos de la memoria final

Parte del material generado durante las pruebas también puede incorporarse al apartado 11. Anexos de la memoria final, como por ejemplo:

  • tablas completas de pruebas realizadas
  • capturas de errores detectados
  • comparativas antes y después de las correcciones

4️⃣ Sugerencias para completar la fase​

Durante la fase de pruebas puedes tener en cuenta algunas recomendaciones:

  • prueba todas las funcionalidades principales de la aplicación
  • intenta utilizar la aplicación como lo haría un usuario real
  • introduce datos incorrectos para comprobar cómo responde el sistema
  • corrige primero los errores que afectan al funcionamiento del sistema

6️⃣ Plantilla de plan de pruebas​

Para organizar las pruebas de tu aplicación puedes utilizar una tabla como la siguiente. Cada fila representa una prueba realizada sobre una funcionalidad de la aplicación.

IDFuncionalidadDescripción de la pruebaResultado esperadoResultado obtenidoComportamiento de la aplicación
P01Registro de usuarioCrear un usuario con datos válidosEl usuario se registra correctamenteCorrectoEl sistema crea la cuenta y redirige al inicio
P02Inicio de sesiónIniciar sesión con email y contraseña incorrectosEl sistema muestra un mensaje de errorError detectadoAparece el mensaje “Usuario o contraseña incorrectos”
P03Formulario de registroEnviar el formulario con campos vacíosEl sistema impide el envío del formularioCorrectoSe muestran mensajes indicando que los campos son obligatorios
🛠Cómo utilizar la tabla
  • ID. Identificador de la prueba (P01, P02, P03, etc.).
  • Funcionalidad. Parte de la aplicación que se está comprobando.
  • Descripción de la prueba. Explica qué acción se realiza en la aplicación.
  • Resultado esperado. Explica qué debería ocurrir si la aplicación funciona correctamente.
  • Resultado obtenido. Indica si la prueba ha sido correcta o si se ha detectado algún error.
  • Comportamiento de la aplicación. Describe qué ocurrió realmente al ejecutar la prueba, especialmente si se detectó algún error.

Se recomienda realizar entre 6 y 10 pruebas, comprobando las funcionalidades principales de la aplicación.
