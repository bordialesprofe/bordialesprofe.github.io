Fase 5 - Despliegue

En esta fase deberás publicar tu aplicación en Internet para que cualquier persona pueda acceder a ella mediante una dirección web.

El objetivo es comprobar que tu proyecto funciona correctamente fuera del entorno de desarrollo y puede utilizarse como una aplicación real.

1️⃣ Qué debes hacer en esta fase​

En esta fase debes publicar tu aplicación en una plataforma accesible a través de una URL.

El objetivo es que cualquier usuario pueda acceder a tu aplicación y utilizar sus funcionalidades básicas.

Durante esta fase deberás:

  • seleccionar una plataforma de despliegue adecuada
  • configurar el proyecto para su publicación
  • verificar que la aplicación funciona correctamente en el entorno real
Aspectos a tener en cuenta en el despliegue

Durante el proceso de despliegue debes comprobar algunos aspectos importantes:

Accesibilidad de la aplicación La aplicación debe ser accesible desde la URL sin errores.

Funcionamiento general Todas las funcionalidades principales deben funcionar correctamente tras el despliegue.

Configuración de servicios externos Si utilizas servicios como bases de datos, autenticación o APIs externas (por ejemplo, Firebase), debes comprobar que están correctamente configurados en el entorno de producción.

Permisos y control de acceso
Debes verificar que:

  • los usuarios solo acceden a sus propios datos
  • el sistema de autenticación funciona correctamente
  • no existen accesos indebidos a la información

Estas comprobaciones permiten asegurar un funcionamiento correcto desde el punto de vista de seguridad y privacidad.

2️⃣ Entregables de la fase​

Al finalizar esta fase deberás entregar:

📄 Entregable 1 – URL del proyecto desplegado​

Dirección web donde se puede acceder a la aplicación.

La aplicación debe estar publicada y funcionando correctamente.

📄 Entregable 2 - Documento breve de despliegue​

Documento en formato PDF donde expliques brevemente:

  • qué plataforma has utilizado para desplegar la aplicación
  • qué pasos has seguido para publicar el proyecto
  • la URL final de la aplicación
  • qué comprobaciones has realizado para verificar que funciona correctamente
  • cómo has configurado los servicios externos y el control de acceso (autenticación, base de datos, permisos)

3️⃣ Qué partes se incorporan a la memoria final​

El trabajo realizado en esta fase se incorporará principalmente al apartado 7. Despliegue de la memoria final.

En este apartado se explicará cómo se ha publicado la aplicación para que pueda utilizarse en Internet. El trabajo desarrollado durante esta fase servirá para documentar cómo se ha pasado del entorno de desarrollo a una aplicación accesible en la web.

Para conocer con más detalle qué debe incluir este capítulo, consulta el apartado Estructura de la memoria final.

Anexos de la memoria final

Parte del material generado durante el despliegue también puede incorporarse al apartado 10. Anexos de la memoria final, como por ejemplo:

  • capturas del proceso de despliegue
  • comprobaciones realizadas tras la publicación

4️⃣ Sugerencias para completar la fase​

Antes de entregar el proyecto:

  • abre la aplicación usando la URL publicada
  • prueba todas las funcionalidades principales
  • revisa que los enlaces funcionan correctamente
  • comprueba que las imágenes y estilos se cargan bien
