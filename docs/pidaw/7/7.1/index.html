7.1 Control de versiones (Git y GitHub)

Durante el desarrollo del proyecto vas a realizar muchos cambios en tu código. A medida que avances, añadirás nuevas funcionalidades, corregirás errores y modificarás partes del proyecto que ya habías implementado. En este proceso es importante contar con una herramienta que te permita registrar estos cambios y mantener un historial del desarrollo del proyecto.

Un sistema de control de versiones es una herramienta que permite guardar distintas versiones de un proyecto a lo largo del tiempo. Cada vez que realizas una modificación en el código puedes registrar ese cambio, lo que permite consultar el historial del proyecto o recuperar versiones anteriores si algo deja de funcionar correctamente.

De forma sencilla, puedes entender el control de versiones como un sistema que va guardando instantáneas del estado de tu proyecto en diferentes momentos del desarrollo. Esto te permitirá volver atrás si cometes un error, comprobar qué cambios se han realizado o continuar trabajando desde versiones anteriores del proyecto.

Representación conceptual del funcionamiento de un sistema de control de versiones.

Además, estas herramientas facilitan el trabajo en equipo, ya que permiten que varias personas trabajen sobre el mismo proyecto sin sobrescribir el trabajo de los demás.

Para gestionar el control de versiones utilizaremos dos herramientas muy habituales en el desarrollo de software:

  • Git, que es el sistema que permite registrar los cambios en el código.
  • GitHub, que es una plataforma web donde se almacenan los repositorios y que permite compartir el proyecto y trabajar de forma colaborativa.

A lo largo del proyecto utilizarás estas herramientas para guardar el código de tu proyecto y mantener un registro de los cambios que vayas realizando.

1️⃣ Git y GitHub​

Para gestionar el control de versiones del proyecto utilizaremos dos herramientas muy habituales en el desarrollo de software: Git y GitHub.

Git es un sistema de control de versiones que permite registrar los cambios realizados en un proyecto. Gracias a Git puedes guardar diferentes versiones del código, consultar el historial de modificaciones y recuperar versiones anteriores si es necesario.

Cuando trabajas con Git, el proyecto se guarda en un repositorio, que contiene todos los archivos del proyecto y el historial de cambios que se han realizado.

Por otro lado, GitHub es una plataforma web que permite almacenar repositorios Git en la nube. Esto facilita compartir el proyecto con otros desarrolladores, trabajar en equipo y mantener una copia remota del código.

En este proyecto utilizarás Git para registrar los cambios en tu código y GitHub para almacenar el repositorio del proyecto y compartirlo con tu equipo.

🔶 Diferencia entre Git y GitHub​

Es importante entender la diferencia entre ambas herramientas:

  • Git es el sistema que se encarga de gestionar las versiones del código.
  • GitHub es la plataforma donde se almacenan los repositorios y se facilita el trabajo colaborativo.

De forma sencilla, puedes pensar que Git es la herramienta que controla las versiones y GitHub es el lugar donde se guardan los proyectos.

2️⃣ Requisitos previos​

Antes de comenzar a trabajar con el control de versiones es necesario realizar una pequeña configuración inicial en tu equipo. Para ello necesitarás instalar algunas herramientas y crear una cuenta en la plataforma que utilizaremos para almacenar los repositorios del proyecto.

🔶 Instalación de Git​

Para poder utilizar un sistema de control de versiones es necesario instalar Git en tu ordenador. Esta herramienta se encargará de registrar los cambios que realices en el código del proyecto y de gestionar las distintas versiones del mismo.

Puedes descargar Git desde su página oficial

Una vez instalado, Git permitirá crear repositorios, registrar cambios en el código y sincronizar el proyecto con el repositorio remoto.

Comprobar si Git está instalado

Para comprobar si Git está instalado en tu equipo, abre una terminal o el símbolo del sistema y ejecuta el comando:

git --version

Si Git está instalado correctamente, se mostrará un mensaje con la versión instalada, por ejemplo: git version 2.45.1

Configuración inicial de Git

La primera vez que utilices Git es recomendable configurar tu nombre de usuario y tu correo electrónico.

git config --global user.name "Tu nombre"
git config --global user.email "tu_correo@example.com"

Configurar estos datos es importante porque permite identificar qué usuario ha realizado cada commit. De esta forma, el historial del proyecto mostrará correctamente el autor de cada cambio.

Esta configuración solo es necesario realizarla una vez en tu equipo.

🔶 Crear una cuenta en GitHub​

También necesitarás disponer de una cuenta en GitHub, ya que utilizaremos esta plataforma para almacenar los repositorios del proyecto y compartir el código con el resto del equipo.

Puedes crear una cuenta gratuita en GitHub.

Para registrarte sigue estos pasos:

  • Accede a la página principal de GitHub.
  • Haz clic en Sign up para crear una nueva cuenta.
  • Introduce tu correo electrónico, un nombre de usuario y una contraseña.
  • Completa el proceso de registro siguiendo las instrucciones que aparecen en pantalla.

Una vez creada la cuenta podrás crear repositorios, subir tu código y colaborar con otros miembros del equipo.

🔶 Visual Studio Code​

Durante el desarrollo del proyecto utilizaremos Visual Studio Code como entorno de desarrollo. Este editor incluye herramientas integradas para trabajar con Git, lo que permite gestionar el control de versiones directamente desde el propio editor.

Desde Visual Studio Code podrás:

  • visualizar los cambios realizados en los archivos
  • realizar commits
  • sincronizar el repositorio con GitHub

todo ello sin necesidad de utilizar la terminal.

3️⃣ Primer repositorio​

Una vez que dispones de una cuenta en GitHub y tienes instalado Git en tu equipo, puedes crear tu primer repositorio y comenzar a registrar los cambios en tu proyecto.

En esta actividad crearás un repositorio en GitHub, lo descargarás en tu ordenador y realizarás tu primer commit.

Proyecto de práctica

Para realizar esta actividad utilizaremos un pequeño proyecto de ejemplo.

Descarga el siguiente proyecto en tu ordenador antes de comenzar con los pasos de la práctica: Repositorio de práctica

Una vez descargado el archivo, descomprímelo en una carpeta de tu ordenador antes de comenzar la práctica.

🔶 Paso 1: Inicializar el repositorio local​

En este punto ya dispones del proyecto de ejemplo en una carpeta de tu ordenador. Sin embargo, todavía no estás utilizando control de versiones sobre ese proyecto.

Para comenzar a trabajar con Git es necesario inicializar un repositorio en la carpeta del proyecto.

Desde Visual Studio Code
  • Abre la carpeta del proyecto en Visual Studio Code.
  • Accede al panel Source Control (icono de ramas).
  • Pulsa el botón Initialize Repository.

Al inicializar el repositorio, Git ha creado un sistema interno que permitirá registrar todos los cambios que realices en el proyecto.

A partir de este momento:

  • Git detectará los archivos modificados
  • podrás guardar versiones del proyecto mediante commits
  • se generará un historial de cambios

Desde consola
También puedes inicializar un repositorio utilizando la terminal.

Para ello, abre una consola en la carpeta del proyecto. Puedes utilizar:

  • Git Bash, haciendo clic derecho sobre la carpeta y seleccionando Open Git Bash here
  • o una terminal del sistema (CMD o PowerShell)

Una vez abierta la consola, ejecuta: git init Este comando convierte la carpeta actual en un repositorio Git.

Comprobar la inicialización del repositorio

Si todo ha ido correctamente, en la carpeta del proyecto se habrá creado una carpeta oculta llamada .git.

Si no la ves, activa la opción de mostrar archivos ocultos en tu sistema.

Esta carpeta contiene toda la información necesaria para que Git pueda gestionar el control de versiones del proyecto.

🔶 Paso 2: Registrar el primer commit​

Una vez inicializado el repositorio, Git comenzará a detectar los cambios que realices en los archivos del proyecto. En este momento, es necesario registrar estos cambios para guardar una primera versión del proyecto.

Para registrar estos cambios, se realiza un commit.

Desde Visual Studio Code

  1. Abre el panel Source Control en Visual Studio Code.
  2. Verás los archivos modificados en la sección de cambios.

  1. Escribe un mensaje de commit que describa el cambio realizado. Por ejemplo: Primer commit
Commit sin mensaje claro

Es importante escribir mensajes de commit claros y descriptivos.

Evita mensajes como "cambios" o "update". En su lugar, describe brevemente qué has hecho.

  1. Pulsa el botón Commit (Confirmación) para guardar los cambios.

Al realizar el commit, Git guarda una nueva versión del proyecto en el historial.

Después de realizar el commit:

  • los cambios quedan registrados en el historial
  • se crea un punto al que podrás volver en el futuro
  • el proyecto tiene ya una primera versión guardada

Desde Consola
También puedes realizar un commit utilizando la terminal.

  • Primero, prepara los archivos con: git add .
  • Después, crea el commit: git commit -m "Primer commit"

El comando git add . selecciona los cambios que quieres guardar, y git commit los guarda en el historial del proyecto.

Ver historial de commits

Puedes consultar el historial de cambios del proyecto.

En algunos casos, Visual Studio Code muestra un apartado llamado Graph, donde se visualizan los commits de forma gráfica.

Si no aparece, puedes utilizar la terminal:

  • git log → historial completo
  • git log --oneline → versión resumida

🔶 Paso 3: Subir el proyecto a GitHub​

Una vez realizado el primer commit, el proyecto sigue estando únicamente en tu ordenador. Para poder compartirlo y trabajar en equipo, es necesario subirlo a GitHub.

Desde Visual Studio Code

  1. Accede al panel Source Control.
  2. Pulsa el botón Publish Branch (Publicar Branch).
  3. Si es la primera vez que utilizas GitHub en Visual Studio Code, se abrirá automáticamente el navegador para iniciar sesión con tu cuenta..
  4. Indica si quieres que el repositorio sea público o privado.

Una vez completado este proceso, el repositorio se creará en GitHub y se subirá automáticamente tu proyecto.

Al pulsar Publish Branch, Visual Studio Code realiza automáticamente varias acciones:

  • crea un repositorio remoto en GitHub
  • conecta tu repositorio local con ese repositorio
  • sube los commits realizados
Si Publish Branch no funciona

En algunos casos, puede ser necesario crear el repositorio manualmente en GitHub.

Para ello:

  1. Accede a GitHub.
  2. Pulsa en New repository.
  3. Introduce un nombre para el repositorio.
  4. Selecciona si será público o privado.
  5. No añadas README ni otros archivos iniciales.
  6. Pulsa en Create repository.

Una vez creado, podrás subir el proyecto desde la terminal.

Desde consola
También puedes subir tu proyecto utilizando la terminal.

Una vez creado el repositorio en GitHub, ejecuta los siguientes comandos en la carpeta del proyecto:

git remote add origin https://github.com/usuario/pidaw-git-practica.git  
git branch -M main  
git push -u origin main

💡 Sustituye la URL por la de tu repositorio en GitHub. Para obtenerla, accede a tu repositorio, pulsa en Code y copia la URL que aparece. Explicación de los comandos:

  • git remote add origin → conecta tu proyecto local con el repositorio de GitHub
  • git branch -M main → establece la rama principal como main
  • git push -u origin main → sube el proyecto a GitHub y guarda la conexión para futuros envíos

El comando git push envía los commits al repositorio remoto en GitHub.

Comprobación

Si todo ha ido bien, podrás ver tu proyecto en tu perfil de GitHub con los archivos y el commit realizado.

👉 Hasta ahora has estado trabajando en un repositorio local. A partir de este momento, tu proyecto también estará disponible en GitHub.

🔶 Paso 4: Realizar cambios y sincronizar el proyecto​

Una vez que el repositorio está en GitHub, el flujo de trabajo será siempre el mismo: modificar archivos, guardar los cambios y subirlos al repositorio remoto.

Desde Visual Studio Code
  1. Realiza algún cambio en un archivo del proyecto (por ejemplo, modifica el contenido de index.html).
  2. Accede al panel Source Control.
  3. Escribe un mensaje descriptivo en el campo de commit.
  4. Pulsa el botón Confirm (✔) para guardar los cambios.
  5. Pulsa Sync Changes para subirlos a GitHub.

En este proceso se realizan dos acciones:

  • Commit → guarda los cambios en tu repositorio local
  • Push (Sync Changes)→ envía esos cambios a GitHub
Desde consola

También puedes realizar este proceso desde la terminal:

git add .
git commit -m "Descripción de los cambios"
git push
  • git add . → prepara los cambios
  • git commit → guarda los cambios
  • git push → los sube a GitHub
Flujo de trabajo habitual

A partir de este momento, cada vez que realices cambios en tu proyecto deberás seguir este proceso:

modificar → commit → push

Error habitual

Si realizas un commit pero no haces push, los cambios no se verán en GitHub.

4️⃣ Sincronizar el proyecto en varios ordenadores​

Una de las principales ventajas de Git y GitHub es la posibilidad de trabajar en el mismo proyecto desde distintos equipos, manteniendo siempre los cambios sincronizados.

El flujo habitual será:

  • descargar el proyecto en un equipo
  • realizar cambios
  • subirlos a GitHub
  • actualizar el proyecto en otro equipo

🔶 Clonar el repositorio​

Si quieres trabajar en otro ordenador, debes descargar el proyecto desde GitHub.

Desde Visual Studio Code

  1. Abre Visual Studio Code.
  2. Pulsa en Clone Git Repository (o accede desde la paleta de comandos).
  3. Pega la URL del repositorio.
  4. Selecciona la carpeta donde quieres guardar el proyecto.
  5. Abre el proyecto cuando finalice la descarga.
Desde consola
  1. Accede a la carpeta donde quieras guardar el proyecto.
  2. Ejecuta: git clone https://github.com/usuario/repositorio.git

Esto creará una carpeta con el nombre del repositorio y descargará todo el proyecto junto con su historial de cambios.

🔶 Subir cambios (push)​

Después de realizar cambios en el proyecto, debes subirlos a GitHub.

Desde Visual Studio Code

  1. Realiza cambios en los archivos.
  2. Accede a Source Control.
  3. Escribe un mensaje de commit.
  4. Pulsa Confirm (✔).
  5. Pulsa Sync Changes para subir los cambios.
Desde consola
git add .
git commit -m "Descripción de los cambios"
git push
  • git add . → prepara los cambios
  • git commit → guarda los cambios en local
  • git push → sube los cambios a GitHub

🔶Obtener cambios (pull)​

Si estás trabajando en varios equipos, es importante descargar los cambios realizados desde otros dispositivos.

Desde Visual Studio Code
  1. Accede a Source Control.
  2. Pulsa Sync Changes o Pull para actualizar el proyecto.
Desde consola
git pull

Este comando descarga y aplica los últimos cambios del repositorio remoto.

Flujo de trabajo en varios equipos

Cuando trabajes en distintos ordenadores, sigue siempre este orden:

  1. git pull → antes de empezar a trabajar
  2. modificar archivos
  3. commit
  4. git push → al terminar
Error habitual

Si no haces git pull antes de trabajar, puedes tener conflictos al subir tus cambios.

👉 GitHub actúa como punto central que permite mantener sincronizado el proyecto entre todos los equipos.
